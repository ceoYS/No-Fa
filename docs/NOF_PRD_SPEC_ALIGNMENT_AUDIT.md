## 1. 문제 요약 (Problem summary)

P0 프로토타입은 실행되지만, 시각 검토에서 **제품 정의 차원의 정렬 결함**이 드러났다.

| 관찰된 결함 | 본질 |
|---|---|
| “My Discipline”에 `지키는 중` 등 상태가 표시되지만 **사용자가 오늘 지켰는지 확인하는 입력 흐름이 없음** | Discipline 모델이 "표시"만 있고 "확인" 동선이 정의되지 않음 — PRD §0.5.9가 비처벌만 정의하고 확인 메커니즘은 미정의 |
| 캘린더가 **장식적**으로 느껴짐 | 캘린더가 무엇을 기록하고 무엇을 보여주는지 (PRD §0.5.3 + FAILENDAR §6.1) 사용자 동선에서 사라짐. P0 미니 7일 스트립만 정의되고 탭 동작이 없음 |
| `조용해진 날` 라벨이 불명확 | 메타포(잔불) 어휘를 1차 라벨로 사용 — PRD §0.5.8 "잔불은 보조 메타포"와 충돌 |
| `월/주 전체 보기는 P1에서 열려요` 같은 **내부 기획 어휘가 사용자 UI에 노출** | PRD §0.5.8 "내부 정책 어휘는 사용자 화면에 노출하지 않는다"와 COPY_POLICY §4.1 위반 |
| PRD §0.5.9는 규율의 카피·금지만 다루며, 실제 **상태 전이/입력/회복 흐름이 미정의** | Spec Kit이 PRD를 충실히 상속하지만, PRD 자체에 행동 모델이 빠져 있어 구현자가 정의하지 못함 |

근본 원인: PRD §0.5.8–§0.5.10이 **표현 정책**(무엇을 노출/금지)은 강하게 정의하지만 **사용자 행동 모델**(어떻게 확인·기록·회복하는가)은 약하게 정의되어 있다. Spec Kit은 PRD를 상속하므로 이 빈 자리를 채우지 않는다. 프로토타입은 그 빈 자리에 들어가 임시 가시화한 결과 정의 충돌이 시각적으로 드러났다.

## 2. PRD 갭 (PRD gaps)

### A. 제품 정의 (Product definition)

| 질문 | PRD 현재 상태 | 갭 |
|---|---|---|
| NoF를 자기통제/충동 보호/회복 앱으로 명료하게 정의하는가? | §1.1 직설형/완곡형 정의 있음. §0.5.8 명료성 절 있음 | **§0.5.8이 §1.1보다 뒤에 묻혀 있어** 첫 독자가 펫·잔불을 코어로 오독할 위험. §1.1을 §0.5.8 톤으로 갱신 필요 |
| 펫/잔불/캘린더를 자기통제/규율/위기/회복보다 과강조하는가? | §0.5.1·§0.5.2·§0.5.3은 펫 코어 루프를 강하게 선언. §0.5.8은 "펫은 보조 메타포" 보정 | **두 문장이 부분 충돌**. §0.5.1 "정서적 코어는 펫 기반 자기통제 회복 루프"와 §0.5.8 "메인 카피는 직접적 자기통제 언어, 잔불은 보조"가 같이 살아 있어 구현자가 어디에 무게를 둘지 모름 |
| 코어 루프가 명확히 설명되어 있는가? | §0.5.1 한 줄, FAILENDAR §3에 다이어그램 있음 | **PRD 본문에 한 화면 한 행동 단위의 일일 루프가 없음**. (매일 진입 → 어떤 행동 → 어떤 상태 변경) |

### B. Discipline 모델

PRD §0.5.9·§0.5.10이 정의하지 못한 것:

1. 사용자가 규율을 **어떻게 정의**하는가 — P0.1 in-memory 추가/삭제만 정의됨. **편집은 P1**로 미뤄짐. P0에서 "지키는 중"이 어떻게 결정되는지 없음.
2. 사용자가 오늘 **지켰는지 확인**하는 입력 액션이 정의되지 않음. (`status`는 enum만 있고 전이 트리거 없음)
3. 상태가 **수동/자동/혼합** 중 무엇인지 미정. 현재 프로토타입은 mock 정적 상태.
4. `recovered`(다시 시도/회복 완료) 상태로의 **전이 경로**가 없음.
5. `weeklyCount` 증가 트리거가 정의되지 않음.

### C. Calendar 모델

PRD §0.5.3·FAILENDAR §6.1이 정의하지 못한 것:

1. P0 미니 7일 스트립이 **무엇을 기록**하는지 정의 불완전. 후보가 흩어져 있음: "흔들림 사건 + 보호 행동 + 체크인 여부"(FAILENDAR §6.1) — 3개 입력의 합산 규칙 없음.
2. 날짜 점이 **무엇을 의미**하는지의 enum이 명시되지 않음. 현재 코드의 `steady`/`quiet`/`unmarked`는 PRD에 없는 어휘.
3. **날짜 탭 시 동작**이 정의되지 않음. 상세 보기 화면은 P1로 미뤄졌으나, P0에서도 탭 피드백/오늘 진입 동선이 필요한지 결정되지 않음.
4. 라벨 어휘가 메타포(`이어온 날`·`조용해진 날`)이며, **사용자 의미 라벨**과 매핑이 없음.

### D. UI 언어 누출

PRD §0.5.8과 COPY_POLICY §4.1은 내부 정책 어휘 노출을 금지한다. 그러나 PRD 본문 자체가 P0/P1, "보기 전용", "코스메틱" 등을 빈번히 사용해 **하위 문서·구현자가 그 어휘를 화면에 노출하기 쉽다**. PRD에 "사용자 화면에서 금지되는 어휘 표"가 한곳에 정리되어 있지 않다.

## 5. 사람이 결정해야 할 항목 (Product decisions needed from human)

PRD/Spec/구현을 진행하기 전 결정이 필요한 항목.

### 5.1 Discipline 확인 모델 (필수, 가장 시급)

D1. 규율 확인 입력 방식:
- (a) 규율별 토글: 행마다 `지켰어요` 버튼
- (b) 일일 일괄 확인: 하루 한 번 "오늘의 규율 확인" 시트에서 각 규율을 4-state로 표시
- (c) 체크인과 통합: 1분 체크인의 한 스텝으로 규율 상태 점검
- 권장: **(c) 체크인 통합 + (a) 행별 부분 갱신 옵션** — 체크인이 P0의 매일 진입 행동이므로 자연스럽다.

D2. 자동/수동 비율:
- 시간 기반 자동 전이 허용 여부 (예: 자정에 `keeping` → `not_recorded` 리셋). PRD §0.5.10 A `lastCheckedAt`이 있음에도 일일 리셋 규칙이 없음.
- 권장: P0는 **사용자 입력 수동만**, P1에서 자동 리셋 도입.

D3. 허용 상태 enum 최종 확정 (질문서 제안):
- 허용: `지켰어요` / `확인 필요` / `흔들렸어요` / `회복 완료` / `오늘은 해당 없음`
- 금지: `위반` / `실패` / `벌점` / `포기` / `점수 차감`
- **PRD §0.5.9·§0.5.10의 현재 어휘**(`keeping` / `needs_check` / `recovered` / `not_recorded` → 한국어 `지키는 중` / `오늘 확인 필요` / `다시 시도` / `아직`)와 일부 차이.
- 결정 필요: **5-state로 확장하고 현 4-state를 매핑할지**, 혹은 4-state를 유지하고 표기만 다듬을지.

D4. `흔들렸어요` 상태와 회복 동선:
- `흔들렸어요` 진입 시 자동으로 회복 화면으로 라우팅할지, 아니면 같은 화면에서 회복 행동 1개 제안만 보일지.

### 5.2 Calendar P0 행동 모델 (필수)

C1. 점이 기록하는 것:
- 후보 a: 체크인 여부만 (점 on/off)
- 후보 b: 체크인 + 흔들림 + 보호 행동 합산 톤 (3축 → 4톤)
- 권장: **(b) 단순화 — 2톤(`지킨 날`/`기록 전`) + 추가 보조 톤 `회복한 날`/`확인 필요`**. (질문서 제안과 일치)

C2. 점 enum 최종 라벨 (질문서 제안):
- 허용: `지킨 날` / `회복한 날` / `기록 전` / `확인 필요`
- 메타포(`이어온 날`/`조용해진 날`) 금지
- 결정 필요: 라벨 채택 여부 + 톤 매핑 (`steady`→`지킨 날` 등)

C3. P0 캘린더 화면 존재 여부:
- (a) 별도 화면 폐기 — FAILENDAR §4 원칙대로 홈 미니 7일만 유지
- (b) 별도 화면 유지하되 "최근 7일" 한 카드만 (월 placeholder 제거)
- (c) 별도 화면 유지하고 기능적 월 뷰 P0 포함 (스코프 확장)
- 권장: **(b)**. 새 화면 추가 금지 원칙과의 충돌은 BottomNav 탭으로 노출되는 P0 화면 추가에 한정하여 PRD §0.5.7 P0 범위 패치로 정리.

C4. 날짜 탭 시 동작:
- (a) 탭 비활성 (장식)
- (b) 탭 시 그 날짜의 체크인 요약 한 줄 표시
- 권장: **(b)** — 캘린더가 "장식적"이라는 시각 검토 결함을 해소.

### 5.3 UI 언어 클린업 (필수)

L1. 다음 어휘를 사용자 UI에서 **전면 금지** 결정 (PRD §0.5.8 보완):
- `P0` / `P1` / `P2` / `보기 전용` / `프로토타입` / `점수` / `위반` / `벌점` / `실패`(상태 어휘로) / `코스메틱` / `enum` / `정책`
- placeholder 카피 통일 어휘는 무엇인가? (예: "준비 중", "다음 단계", "곧 열려요" 중 하나로 통일)

### 5.4 PRD 위계 정리 (선택)

H1. PRD §0.5.1과 §0.5.8의 강조 충돌 해소:

## 7. Discipline P0 제안 동작 (Proposed P0 behavior for Discipline)

> 본 절은 사람 결정 (§5.1) 전 **권장안**이다.

### 7.1 상태 모델 (제안)

- 허용 상태 (5-state, 사용자 표시 어휘):
  - `지켰어요` (오늘 분명히 지킴)
  - `확인 필요` (오늘 아직 점검하지 않음 — 비난 어휘 아님, 기본값)
  - `흔들렸어요` (오늘 지키지 못함 — "실패" 어휘 회피)
  - `회복 완료` (흔들린 뒤 회복 행동 1개 수행)
  - `오늘은 해당 없음` (예: 출근하지 않는 날에 "잠들기 전 휴대폰" — 적용 안 됨)
- 금지 어휘: `위반` / `실패` / `벌점` / `포기` / `점수 차감`
- 내부 매핑 (PRD §0.5.10 B 점수 — 사용자에게 비공개):
  - `지켰어요` = 1.0, `회복 완료` = 0.8, `흔들렸어요` = 0.4, `확인 필요` = 0.6 중립, `오늘은 해당 없음` = (계산 제외)

### 7.2 확인 입력 동선 (제안)

- **체크인 마지막 스텝**에 "오늘의 규율 점검" 카드 추가 (1분 체크인 한도 안에서):
  - 각 규율 행: 라벨 + `지켰어요` / `흔들렸어요` / `해당 없음` 3-tap 칩
  - 미선택 규율은 자동으로 `확인 필요`로 유지
- Discipline 화면에서는 **부분 갱신** 허용:
  - 행별 `· · ·` 액션 → 상태 변경 시트 (5-state 칩)
  - `흔들렸어요` 선택 시 → 카드 하단에 "회복 행동 1개 해볼까요?" 한 줄 + 회복 화면으로 보조 CTA
  - 회복 행동 완료 후 → 자동으로 `회복 완료` 전이

### 7.3 비처벌 매핑 (PRD §0.5.5·§0.5.10 E 상속)

- `흔들렸어요` 누적 → 펫 강등 0, 잔불 약화(`ember_low`)만 허용
- 사용자에게 카운트/숫자/점수 비표시 — "이번 주 흔들림 2회" 같은 빈도 표기도 카드 보조 노트에만 (회복 톤)
- 자동 공유 0 (§8 상속)

### 7.4 P0 vs P1 경계

- **P0**: 5-state 표시 + 체크인 통합 확인 입력 + 행별 부분 갱신 + 회복 보조 CTA + in-memory 추가/삭제
- **P1**: 영속화, 본격 편집 UI, 템플릿, 위험 시간 리마인더 연동, 규율별 인사이트 (PRD §0.5.9와 일치)

## 8. Calendar P0 제안 동작 (Proposed P0 behavior for Calendar)

### 8.1 점이 기록하는 것 (제안)

P0 캘린더 점은 **오늘 하루의 자기통제 상태 한 가지**를 표시한다. 합산 규칙:

| 입력 (그 날) | 점 상태 |
|---|---|
| 체크인 완료 + 흔들림 없음 | `지킨 날` |
| 흔들림 있음 + 회복 행동 1개 이상 완료 | `회복한 날` |
| 흔들림 있음 + 회복 행동 0 또는 규율 `확인 필요` 있음 | `확인 필요` |
| 그 날 어떤 기록도 없음 | `기록 전` |

### 8.2 라벨/어휘 (제안)

- 허용 어휘: `지킨 날` / `회복한 날` / `기록 전` / `확인 필요`
- 메타포 금지: `이어온 날` / `조용해진 날` / `잔불 캘린더`(화면 타이틀은 더 직접적으로) — `최근 기록` 또는 `절제 기록`
- 톤 매핑 (코드 enum 변경 권장):
  - `kept` (지킨 날) — 따뜻한 앰버
  - `recovered` (회복한 날) — 부드러운 앰버 + 가벼운 링
  - `needs_check` (확인 필요) — 잔잔한 톤 + 미세 점선
  - `untracked` (기록 전) — 점선 윤곽만

### 8.3 화면 구성 (제안 — §5.2 C3 (b) 채택 가정)

- 홈: 최근 7일 미니 스트립 + "전체 보기" 보조 CTA
- 별도 캘린더 화면: 최근 7일 카드 한 장 + 각 점 **탭 시 한 줄 요약** (예: "5월 24일 — 지킨 날 · 체크인 완료, 흔들림 없음")
- **월 placeholder 제거** — "월/주 전체 보기는 P1" 카드 자체 삭제. 필요하면 설정 → 도움말에 한 줄 안내.

### 8.4 비행동 규칙

- 빨강/이진 금지 (FAILENDAR §2 상속)
- 점수/등급/퍼센트 비표시 (PRD §0.5.10 D 상속)
- 자유 서술 비강제 (UX_RULES §10 상속)
