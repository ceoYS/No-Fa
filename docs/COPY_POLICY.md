# COPY_POLICY.md

Korean NoFap / 절제 앱 — 카피 정책

상위 문서: PRD §1.2, §16 / 같은 라운드: UX_RULES.md, SCREEN_INVENTORY.md
이 문서는 카피 작성·검수 기준이다. 실제 화면 카피 양산은 디자인/프로토타입 단계에서 이 문서를 근거로 진행한다.

---

## 1. 카피 미션 (Copy mission)

수치심·종교·의료·모욕·과금 압박 없는 카피로, **회복과 자기통제**를 지지한다.
민감 주제이므로 AI는 임의로 직설적/자극적 카피를 만들지 않는다.

---

## 2. 승인 어휘 (Approved vocabulary)

기본 사용 (앱 내부·앱스토어·온보딩):
- 절제 / 자기통제 / 회복 / 루틴
- 유해 디지털 환경 / 내가 정한 환경
- 잠깐 멈춤 / 다시 시작 / 함께 버팀

프레이밍: "내가 정한 환경을 지킨다", "다시 시작은 회복의 시작점".

---

## 3. 제한 어휘 (Restricted vocabulary)

위치·맥락 제한 사용. 앱 내부 기본 카피로는 지양, 검색/유입/커뮤니티 맥락에서만 제한적으로.
- 금딸 (검색 광고·커뮤니티 유입·SEO 실험 한정 — PRD §1.2)
- NoFap (커뮤니티/콘텐츠 유입 한정)
- 충동 / 유해 사이트 / 차단

규칙: 제한 어휘는 앱스토어 노출/온보딩 본문 기본 카피로 쓰지 않는다. 사용 시 사람 검수.

---

## 4. 금지 어휘 (Forbidden vocabulary)

어디에서도 사용 금지:
- 죄 / 타락 / 실패자 / 중독자
- 치료된다 / 우울증이 낫는다 / 불안이 사라진다
- 꼴찌 / 부끄럽지 않나요
- 절대 못 봅니다 / 완벽 차단 / 절대 우회 불가
- 돈 내면 실패 복구 (및 유료 실패 삭제·스트릭 복구·차단 해제권 암시)
- 규율 위반 / 벌점 / 위반 횟수 (PRD §0.5.9 — 1차 사용자 카피로 금지)

### 4.1 사용자 화면 노출 금지 어휘 (Internal-policy / planning phrasing)

PRD §0.5.8·§0.5.8.1 기준. 다음은 운영/기획 어휘일 뿐이며 **사용자 UI 카피로 어디에서도 노출하지 않는다.** (화면, 모달, 빈 상태, placeholder, 캘린더 라벨, 상태 칩, 토스트 모두 포함)

**내부 단계 라벨**
- `P0`, `P1`, `P2`, `P0.1`, `P0.5`, `MVP`, `프로토타입`, `보기 전용`, `데모`, `mock`
- "월/주 전체 보기는 P1에서 열려요" 류 단계 placeholder 카피 금지
- "다음 단계에서 열어요 (P1)" 류 금지 → 필요 시 "곧 열려요" 같은 일반 어휘로

**운영/정책 어휘**
- `중립 표기 · 처벌 없음`, `코스메틱·케어 기반 보상`, `결제 없음 / 뽑기 없음 / 한 번 더 뽑기 없음`
- `능력치`, `확률 보상`, `pay-to-win`
- `enum`, `bucket`, `allowlist`, `정책` — 사용자 카피로 노출 금지
  - "enum으로만 기록돼요" 금지 → **"선택지로만 기록돼요"** 사용
  - "자유서술은 받지 않아요" 금지 표현 아니지만 "직접 쓰지 않아도 괜찮아요" 톤 권장

**평가·처벌 어휘**
- `위반`, `규율 위반`, `위반 횟수`, `실패`(상태 의미로), `실패자`, `벌점`, `점수 차감`, `강등`, `등급`, `랭킹`, `점수`, `순위`, `포기`
- 점수 부재 강조 카피("깎이는 점수는 없어요", "벌점 없어요") **금지** — 점수 프레임 자체를 환기시킴
- **사회 기능 단계 도입과 무관하게 유지(§0.6.8):** 사회적 동행이 P0 티저 → P1 기능 → P2/P3 리그로 단계 도입되더라도, `랭킹`·`등급`·`점수`·`순위`·`강등`은 **사용자 노출 어휘로 계속 금지**한다. 동행은 "함께 버티는"·"같이 가는 사람들"로만 표현한다. 경쟁 어휘를 사용자에게 허용하려면 본 §4.1을 **명시적으로 개정**해야 한다(§0.6.8 remaining decision). 수치심 랭킹·재발 공개 노출·강제 소셜 공유도 단계와 무관하게 금지.

**메타포 단독 1차 카피**
- 화면 1차 타이틀로 `잔불 캘린더` 금지 → `최근 기록` 또는 `절제 기록`
- 캘린더 라벨로 `이어온 날`, `조용해진 날`, `아직` 금지 → §10.6 승인 어휘 사용
- "잔불"·"잔불이 이어지는 중" 같은 메타포 단독 카피를 1차 카피로 사용 금지(보조 메타포로만)

**비공개 라벨 남발**
- `비공개 기록` / `나만 보는 기록`을 핵심 메트릭 카드 반복 배지로 노출 금지 (§10.1 상속)

## 5. 앱스토어 문구 정책 (App Store wording policy)

PRD §4 (Option A 타깃 / Option B 표현) 기준.
- 성적 표현 최소화. "절제", "자기통제", "유해 디지털 환경 차단" 사용.

- 성인 콘텐츠 제공 앱으로 오해될 표현/사이트명 금지.
- 미성년자 대상 마케팅 문구 금지.

## 6. 인앱 문구 정책 (In-app wording policy)

- 톤: 담담·비판 없음. 위기엔 짧고 단호, 회복엔 따뜻.
- 명령형("참아라") 대신 제안형("5분만 다른 행동 해볼까요?").
- 자기 진단 강요("얼마나 심각한가요") 금지.
- 자유서술 입력은 선택·비공개. 강요 금지.

## 8. 실패/회복 카피 정책 (Failure/recovery copy policy)

PRD §0.6.3(실패 범위) + §0.6.6(복기 다이어리) 기준. **두 실패 범위를 카피에서 구분한다.** 둘 다 비난·수치심 없이 회복 루프로 연결한다.

**두 범위 (카피 톤 분리):**

| 범위 | 사용자 표현 | 타이머 | 복기 | 톤 |
|---|---|---|---|---|
| 재발 (금욕 실패) | `다시 시작` / `무너졌어요` | 리셋(0일) | 필수 | 담담·재출발. "다시 시작은 회복의 시작점" |
| 규율 슬립 | `못 지켰어요` | 리셋 안 함 | 가벼움(선택) | 가벼운 환기. "잠깐 돌아볼까요?" |

- 재발=여정의 끝이 아니라 회복 루프 시작. "여정이 끝난 건 아니에요" 톤. 타이머가 0으로 돌아가도 비난 어휘 금지.
- 규율 슬립은 재발과 다르며 타이머를 리셋하지 않는다. "못 지켰어요"는 무거운 실패가 아니라 가벼운 점검 신호로 카피한다(§10.5 상속).
- "또 실패", "의지가 약해서", "결국 또" 류 금지.
- 복기 기록은 처벌이 아니라 다음 패턴 학습으로 프레이밍. 복기 완료 후엔 따뜻하게: "복기 완료. 오늘도 다시 이어가요."

**복기 다이어리 프롬프트 카피 (§0.6.6, 3문항):**

- 재발 시 표시(필수 입력 권장, 그러나 강요 금지): `왜 그랬을까요?` · `무엇이 계기였어요?` · `다음엔 어떻게 해볼까요?`
- `위기였지만 버텼어요` 시: 동일 프롬프트를 **선택**으로 제시("기록해두면 다음에 도움이 돼요"). 강제하지 않는다.
- 프롬프트는 질문형·비판 없음. "왜 못 참았나요" 류 추궁 금지 → "무엇이 계기였어요?"로.
- 복기 텍스트는 비공개(§10.6 상속). 외부·파트너·점 요약에 노출하지 않는다.

### 재발/복기 좋은 카피 예시 (Good)

1. "다시 시작할 준비가 됐어요." (타이머 리셋 화면)
2. "무너져도 여정이 끝난 건 아니에요. 잠깐 돌아볼까요?"
3. "무엇이 계기였어요?" (복기 프롬프트)
4. "다음엔 어떻게 해볼까요?" (다음 행동 프롬프트)
5. "복기 완료. 오늘도 다시 이어가요."

### 재발/복기 나쁜 카피 예시 (Bad)

1. "또 무너졌네요." → 비난
2. "의지가 약해서 그래요." → 모욕
3. "왜 못 참았나요?" → 추궁
4. "스트릭이 0이 됐습니다. 처음부터 다시." → 처벌 프레이밍
5. "이번에도 실패했어요." → 실패 상태 어휘(§4.1)

## 10. 파트너 공유 카피 정책 (Partner sharing copy policy)

PRD §8 기준.
- 동의 기반 응원자. "감시" 단어 금지.
- 공유 범위는 사용자가 직접 선택. 기본 비공개 명시.
- 실패 여부/원인 공유 권유 금지.

### 10.1 비공개 라벨 사용 규칙 (Privacy label usage)

기본 원칙: NoF 기록은 기본 비공개. UI에 "비공개 기록"·"나만 보는 기록"을 반복 배지로 노출하지 않는다.

- 홈/대시보드의 핵심 메트릭에 "비공개 기록"·"나만 보는 기록" 1차 노출 금지. 대신 가치 기반 라벨을 쓴다: "절제 기록", "최근 기록", "오늘 보호한 자극", "나의 규율".
- 비공개 안내는 온보딩 · 설정 · 도움말에서 1회로 충분히 설명한다.
- 입력형 화면(체크인 등)은 짧은 한 줄로만 안심을 준다: "공유하지 않는 한 기록은 밖으로 나가지 않아요."
- 같은 안내 문구를 여러 화면에서 반복하지 않는다.
- 공유/내보내기/파트너 전송 직전에는 미리보기와 명시 동의 카피를 사용한다:
  - "이 내용을 공유할까요?"
  - "공유 전 내용을 확인하세요."
- P0에는 공유/게시/공개 피드 기능 자체가 없다. 미래의 공유는 항상 명시적·동의 기반·미리보기 기반이다.

## 10.5 나의 규율 카피 정책 (My Discipline copy policy)

PRD §0.5.9 + **§0.6.2(3-state 대체)** 기준. 규율은 사용자가 직접 정한 사적 자기통제 기준이며 비공개·비처벌·비감시·회복 우선이다.

**승인 선택 상태 어휘 (3-state, PRD §0.6.2):**

| 사용자 어휘 | 의미 |
|---|---|
| `지켰어요` | 그 날 규율 지킴 |
| `못 지켰어요` | 그 규율 실패 → 가벼운 복기 유도(**타이머 리셋 안 함**) |
| `위기였지만 버텼어요` | 충동/위기 있었으나 실패 안 함 — **성취** |

**행동 후 자동 배지 (선택 불가):** `복기 완료` · `회복 루틴 완료` · `다음 행동 작성 완료`. 해당 행동을 한 뒤에만 표시한다.

**제거된 어휘:** `해당 없음` · `상황 없음` · `확인 필요`(라벨) · 직접 선택 `회복 완료`. 미선택 규율은 라벨 없이 3버튼 미선택으로 둔다. 스킵이 필요하면 보조 액션 `오늘 기록 건너뛰기`만.

상태 표시형으로만 쓴다("오늘 지켰어요", "위기였지만 버텼어요"). 명령형("지켜야 한다") 금지.

**금지 어휘 (§4.1 상속):** `위반`, `규율 위반`, `위반 횟수`, `실패`(상태로), `벌점`, `점수 차감`, `포기`, `강등`, `등급`, `랭킹`, `점수`.

**기타 규칙:**

- 규율은 "약속"·"맹세"·"공약" 같은 공개 약속 어휘로 부르지 않는다. "내가 정한 기준"으로 표기한다.
- 규율은 점수·랭킹·등급으로 환산하지 않는다.
- 규율 미지킴(`못 지켰어요`)을 펫 처벌·캐릭터 강등으로 연결하지 않으며, **절제 타이머를 리셋하지 않는다**(리셋은 재발만, §0.6.3).
- `위기였지만 버텼어요`는 부정이 아니라 버텨낸 성취로 카피한다("위기였지만 버텼어요").
- 규율의 외부 자동 공유 어휘를 쓰지 않는다 (파트너 공유는 P2 + 옵트인, §10 상속).
- 규율 삭제 카피는 회복 우선. 실패·포기 프레이밍을 쓰지 않는다.
  - 권장: "이 규율을 삭제할까요?", "삭제해도 언제든 다시 추가할 수 있어요."
  - 금지: "이 규율을 포기할까요?", "이 규율을 포기하면 …" 류.
- 점수 부재를 반복 환기하는 카피("깎이는 점수는 없어요" 류) 금지 — 점수 프레임 자체를 떠올리게 함.
- "선택지로만 기록돼요" 권장. `enum` 어휘 노출 금지 (§4.1).

### 좋은 카피 예시 (Good)

1. "내가 정한 기준을 오늘도 지켰어요."
2. "위기였지만 버텼어요 — 잘 버텨냈어요."
3. "오늘은 못 지켰지만, 잠깐 돌아볼까요?" (가벼운 복기 유도)
4. "복기 완료. 오늘도 다시 이어가요."
5. "이번 주 4회 실천"
6. "선택지로만 기록돼요. 직접 쓰지 않아도 괜찮아요."

### 나쁜 카피 예시 (Bad)

1. "규율을 위반했습니다." → 처벌 프레이밍
2. "오늘도 실패했어요." → 비난
3. "벌점 -10" → 점수화
4. "친구에게 규율 어김을 알릴까요?" → 자동 감시
5. "공개 맹세에 서명하세요." → 공개 약속화
6. "깎이는 점수는 없어요." → 점수 프레임 환기
7. "enum으로만 기록돼요." → 운영 어휘 노출
8. "오늘은 해당 없음" / "상황 없음" → 제거된 모호 상태(§0.6.2)

## 10.6 기록 캘린더 카피 정책 (Records Calendar copy policy)

PRD §0.5.3·§0.5.3.2 + **§0.6.5(기능적 일 원장)** 기준. 캘린더는 장식 점이 아니라 **기능적 일 원장**이다. 날짜를 탭하면 그 날 DayRecord 상세를 보여주고, 기간 선택(7/14/30/이번 달)을 제공한다.

**승인 라벨 (점 상태):**

| 라벨 | 의미 |
|---|---|
| `지킨 날` | 체크인 완료 + 흔들림 없음 |
| `회복한 날` | 흔들림 있음 + 회복 행동 1개 이상 완료 |
| `확인 필요` | 흔들림 있음 + 회복 행동 0, 또는 규율 미점검 있음 |
| `기록 전` | 그 날 어떤 기록도 없음 |

**승인 화면 타이틀:** `최근 기록` 또는 `절제 기록`. **승인 기간 라벨:** `최근 7일` / `최근 14일` / `최근 30일` / `이번 달`.

**금지 라벨 (§4.1 상속):**

- `이어온 날`, `조용해진 날`, `아직` — 메타포 단독 라벨 금지
- `잔불 캘린더` — 화면 1차 타이틀로 금지(내부 메타포 명칭으로만)
- 빨강/이진(성공·실패)·실패 마커(빨강 X) 톤 금지 — 재발일은 `다시 시작한 날`/dimmed로
- "월/주 전체 보기는 P1에서 열려요" 류 단계 placeholder 카피 금지 — **기간 뷰는 실기능이므로 동작해야 한다**
- "이번 달 (보기 전용)" 류 **동작 안 하는** placeholder 카피 금지

**점/요약(strip)에는 한 줄 enum 신호만, 자유서술은 상세 시트에만:**

- 점 위 요약 줄은 enum 신호만: "5월 24일 — 지킨 날 · 체크인 완료, 흔들림 없음"
- 날짜 탭 → **DayRecord 상세 시트**에서 그 날 전체를 보여준다: 절제 상태 · 스트릭 일수 · 지킴/미지킴/버팀 집계 · 사유 · 트리거 · **사용자 본인의 복기 텍스트(비공개)** · 다음 행동 · 배지
- 상세 시트의 복기/사유는 **비공개**다. 외부·파트너에 자동 공유하지 않으며, 점 요약 줄·홈에는 노출하지 않는다(UX_RULES §10.7 상속).

## 10.7 사회적 동행 카피 정책 (Companionship copy policy)

PRD §0.6.8(사회적 동행 단계) + UX_RULES §9 기준. 사회 기능은 P0 티저 → P1 기능 → P2/P3 리그로 단계 도입되지만, **카피는 경쟁이 아니라 동행**이다. 비교·서열·노출이 아니라 "혼자가 아니다"라는 신호만 전한다.

**프레이밍 원칙:**

- 동행은 "함께 버티는"·"같이 가는 사람들"로만 표현한다. 사용자를 서로 비교하거나 순위로 줄 세우지 않는다.
- P0 티저는 존재감(presence) 신호만: 지금 함께 버티는 사람이 있다는 사실. 개인 식별·실명·재발 여부 노출 금지.
- 익명·집계 기반. 특정 개인의 재발/스트릭/실패를 다른 사용자에게 보여주지 않는다.

**금지 어휘 (§4.1 상속, 단계 무관):** `랭킹` · `등급` · `점수` · `순위` · `강등` · `꼴찌` · `상위 N%`(서열 노출형). 경쟁 어휘를 사용자에게 노출하려면 §4.1을 **명시적으로 개정**해야 한다(§0.6.8 remaining decision).

**금지 패턴 (단계 무관):**

- 수치심 랭킹·서열표·리더보드형 개인 비교 노출
- 재발/실패의 공개 노출 (누가 무너졌는지 타인에게 표시)
- 강제 소셜 공유 (옵트인 없는 자동 게시·자랑)
- pay-to-win·뽑기·등급 구매로 동행 지위를 사는 카피

### 동행 좋은 카피 예시 (Good)

1. "지금 함께 버티고 있는 사람들이 있어요."
2. "혼자 버티지 않아도 됩니다."
3. "같이 가는 사람들 — 오늘도 각자의 자리에서."
4. "오늘 함께 멈춘 사람: 여럿"  ← 익명·집계 신호

### 동행 나쁜 카피 예시 (Bad)

1. "전체 12위 / 강등 위험" → 서열·강등(§4.1)
2. "상위 5%에 들었어요" → 서열 노출
3. "○○님이 어제 무너졌어요" → 재발 공개 노출
4. "친구에게 내 스트릭을 자랑하기" → 강제·자랑 프레이밍
5. "프로 등급을 사면 리그 1부로" → pay-to-win

## 11. Paywall 카피 정책 (Paywall copy policy)

MONETIZATION_POLICY.md 기준.
- 프로=더 세밀한 분석을 원하는 사용자의 선택.
- "무료는 부족/제대로 관리 못 함" 금지.
- 위기 지원·회복을 유료로 묘사 금지.
- "한 번 더 뽑기", 과금 압박 금지.

## 12. 한국어 톤 예시 (Korean tone examples)

### 좋은 카피 (Good)
1. "잠깐 멈춰도 괜찮아요."
2. "지금은 선택을 늦추는 시간이 필요해요."
3. "5분만 다른 행동을 해볼까요?"
4. "차단은 벌이 아니라, 내가 정한 환경을 지키는 장치예요."
5. "기록이 끊겨도 여정이 끝난 건 아니에요."
6. "다시 시작은 회복의 시작점이에요."
7. "혼자 버티지 않아도 됩니다."
8. "공유 범위는 직접 정할 수 있어요."

### 나쁜 카피 (Bad)
1. "또 실패했네요." → 비난
2. "의지가 약해서 그래요." → 모욕
3. "당신은 중독자입니다." → 낙인
4. "치료 효과가 있습니다." → 의료 단정
5. "신 앞에서 부끄럽지 않나요?" → 종교/수치심
6. "꼴찌입니다." → 비교/망신
7. "절대 우회할 수 없습니다." → 보장성 과장
8. "성인 사이트 목록을 확인하세요." → 성인 콘텐츠 노출
9. "무료 사용자는 제대로 관리할 수 없습니다." → 과금 압박
10. "돈을 내면 실패를 없앨 수 있어요." → pay-to-win

### 재작성 예시 (Rewrite)
- "또 실패했네요." → "다시 시작할 준비가 됐어요."
- "참아야 한다." → "5분만 다른 행동을 해볼까요?"
- "완벽하게 차단됩니다." → "내가 정한 환경을 지키도록 도와요. (모든 경로를 막을 수는 없어요.)"
- "무료는 부족합니다." → "더 세밀한 분석이 필요하면 프로를 선택할 수 있어요."

## 13. 카피 검수 체크리스트 (Copy review checklist)

- [ ] 금지 어휘(§4)가 0건인가?
- [ ] 제한 어휘(§3)가 허용 위치에만 쓰였는가?
- [ ] 치료/완치 단정이 없는가?
- [ ] "완벽 차단/절대 우회 불가" 보장이 없는가?
- [ ] 수치심·종교·모욕·비교 프레이밍이 없는가?
- [ ] 성인 사이트명/URL/검색어가 없는가?
- [ ] 과금 압박·pay-to-win 암시가 없는가?
- [ ] 파트너 공유가 동의 기반/기본 비공개로 표현됐는가?
- [ ] 위기/회복 카피가 무료·회복 중심인가?
