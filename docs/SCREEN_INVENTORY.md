# SCREEN_INVENTORY.md

Korean NoFap 앱 — 화면 목록 (디자인·구현 전 범위 고정)

상위 문서: PRD §7, §8.1, §13, §14, §0.6 / 같은 라운드: UX_RULES.md, COPY_POLICY.md
이 문서는 화면 **범위**를 고정한다. 화면 디자인·와이어프레임 생성은 하지 않는다.

> **v0.3.2 코어 리포커스(PRD §0.6) 반영:** 코어 히어로는 **절제 타이머**(재발만 리셋), Discipline은 **3-state**(`지켰어요` / `못 지켰어요` / `위기였지만 버텼어요`) + 행동 후 자동 배지(`복기 완료` / `회복 루틴 완료` / `다음 행동 작성 완료`), 캘린더는 **DayRecord 기능 원장**(날짜 탭 → 상세 + 기간 선택 7/14/30/이번 달), 실패/회복 화면은 **복기 다이어리**, 사회는 **동행 티저**(P0 비기능). `해당 없음`·`상황 없음`·`확인 필요`·직접 선택 `회복 완료`는 제거. 충돌 시 §0.6 > §0.5 > 본문.

---

## 1. 화면 목록 목적 (Screen inventory purpose)

- 디자인·프로토타입·구현 전에 MVP 화면 범위를 고정한다.
- 화면별 목적·CTA·금지 카피·수락 기준을 명시해 AI 임의 해석을 막는다.
- P0 / P0.5 / P1 / P2 경계를 명확히 한다. **새 화면 추가 금지** (PRD 변경 요청 선행).
- **단계 어휘는 운영 문서 어휘다.** 단계 어휘(`P0`, `P1`, `P2`, `MVP`, `프로토타입`, `보기 전용`)는 본 인벤토리·UX_RULES·COPY_POLICY 안에서만 쓰며, **사용자 UI 카피·placeholder·빈 상태로 노출하지 않는다** (PRD §0.5.8.1). 미래 기능이 P0에서 아직 없다면 placeholder를 만들지 않거나, 일반 어휘("곧 열려요")로만 표현한다.

화면 정의 항목: 화면명 / 우선순위 / 목적 / 사용자 상태 / 핵심 컴포넌트 / 주요 CTA / 보조 CTA / 표시 데이터 / 금지 카피 / 수락 기준 / Mobbin 벤치마크 수집.

## 2. P0 화면 (P0 screens — MVP Core Loop)

### Splash
- 우선순위: P0 / 사용자 상태: 첫 진입
- 목적: 브랜드 첫인상, 무겁지 않은 톤
- 핵심 컴포넌트: 로고, 짧은 문구 / 주요 CTA: 시작하기 / 보조 CTA: 없음
- 표시 데이터: 없음
- 금지 카피: "중독 치료", 치료/완치 단정
- 수락 기준: 무겁지 않은 톤, 3초 내 다음 진입 가능
- Mobbin 수집: `01_Onboarding`

### 온보딩 (Onboarding)
- 우선순위: P0 / 사용자 상태: 기대·민망함
- 목적: 문제·목표 파악, 수치심 최소화
- 핵심 컴포넌트: 3~5개 질문(enum 선택), "내가 무너지는 경로" 선택 / 주요 CTA: 다음 / 보조 CTA: 건너뛰기
- 표시 데이터: 선택지(enum)
- 금지 카피: "얼마나 심각한가요", 성적 고백 강요
- 수락 기준: 완료율 가설 70%, 자유서술 비강제
- Mobbin 수집: `01_Onboarding`

### 목표 설정 (Goal Setup)
- 우선순위: P0 / 사용자 상태: 결심
- 목적: 현실적 목표 설정
- 핵심 컴포넌트: 기간(7/14/30/커스텀), 이유, 알림 설정 / 주요 CTA: 목표 만들기 / 보조 CTA: 나중에
- 표시 데이터: goal_days bucket
- 금지 카피: "실패 금지", 압박형
- 수락 기준: 설정률 가설 80%
- Mobbin 수집: `01_Onboarding`, `04_Habit`

### 홈 대시보드 (Home Dashboard)
- 우선순위: P0 / 사용자 상태: 일상 진입
- 목적: 오늘 상태 요약, 명확한 오늘 행동
- 핵심 컴포넌트 (우선순위 상→하, PRD §0.6.4):
  - 헤더(아이브로우 "오늘도 지키는 중이에요" + 타이틀 "절제 시간")
  - **절제 타이머 카드 (히어로)** — 마지막 재발 이후 실시간 경과(일 + 시·분·초 카드형). `최장 기록`은 이 카드에 **파생값으로 통합**. 헤더 H1 폭주 방지. **하드코딩된 중복 스트릭 카드(현재/최장 별도 카드)는 제거**(PRD §0.6.1·§0.6.4)
  - **오늘의 규율 점검 진입** — 3-state 점검으로 들어가는 카드/버튼(PRD §0.6.2)
  - **실패/복기 기록 진입** — 차분한 `무너졌어요 / 다시 시작` + `복기하기` (재발 = 타이머 리셋 + 필수 복기, PRD §0.6.3)
  - **충동 멈춤 진입** ("못 참을 것 같아요") — `위기였지만 버텼어요`로 연결
  - 나의 규율 요약 카드 (요약 한 줄 + 보조 한 줄 + 최대 2개 컴팩트 규율 row + "규율 보기" CTA, 전체 리스트는 DisciplineScreen에서, PRD §0.5.9)
  - 최근 기록 미리보기 (최근 7일 미니 스트립 — 각 점 **탭 가능 → 그 날 상세**, PRD §0.6.5)
  - **사회적 동행 티저** ("함께 버티는 중" — P0 비기능, 동행 프레이밍만, PRD §0.6.8)
  - 펫 스테이지(EmberCat) + 오늘의 방 온기(약함 / 잔잔함 / 안정 / 따뜻함, 숫자 비표시, **단일 소스 파생** §0.5.10 C) + 오늘 보호한 자극 — **보조 스트립으로 강등**
- 주요 CTA: 오늘 규율 점검하기 / 보조 CTA: 무너졌어요·복기하기, 충동 멈추기, 규율 보기, 전체 기록 보기
- 표시 데이터: 절제 경과 시간(일·시:분:초, 마지막 재발 기준), 최장 기록(파생), character_stage, 오늘 체크인 여부, 오늘 보호한 자극 count(이미지/영상 mock count), discipline 상태(**3-state: 지켰어요 / 못 지켰어요 / 위기였지만 버텼어요** + 행동 후 배지 복기 완료 / 회복 루틴 완료 / 다음 행동 작성 완료 — PRD §0.6.2), room_warmth 상태(약함 / 잔잔함 / 안정 / 따뜻함 — 내부 계산값은 비공개), 최근 7일 점 상태(지킨 날 / 회복한 날 / 확인 필요 / 기록 전, PRD §0.5.3.2), 동행 존재감(익명 집계 숫자만, 리더보드 아님)
- 금지 카피: "참아라", 명령형, **내부 단계 어휘(P0/P1/P2/MVP/프로토타입/보기 전용)**, 운영 어휘(예: "중립 표기·처벌 없음", "코스메틱·케어 기반 보상", "결제·뽑기 없음", "enum", "정책"), "규율 위반"·"실패"·"벌점"·"포기" 류 1차 카피, 방 온기/규율의 **숫자·점수·퍼센트 노출 금지** (§0.5.10), 점수 부재 강조("깎이는 점수는 없어요") 금지, **"나만 보는 기록"·"비공개 기록"을 핵심 메트릭의 반복 배지로 노출 금지** (기록 비공개는 기본값 — UX_RULES §10.1 / COPY_POLICY §10.1), 메타포 단독 라벨("이어온 날", "조용해진 날", "아직", "잔불 캘린더") 금지 (PRD §0.5.8.1)
- 수락 기준: 오늘 할 행동 CTA 명확, **자기통제/충동 보호/회복 앱으로 명료히 읽힘** (PRD §0.5.8), 잔불·펫은 보조 메타포로만 사용, 타이머가 H1을 폭주시키지 않음, 정량 모델은 상태 어휘로만 번역됨(§0.5.10 D), 비공개 라벨은 홈의 반복 배지로 쓰지 않고 가치 기반 라벨("절제 기록", "최근 기록", "오늘 보호한 자극", "나의 규율") 사용
- Mobbin 수집: `04_Habit`, `05_Streak`

### 스트릭 카드 (Streak Card)
- 우선순위: P0 / 사용자 상태: 동기 확인
- 목적: 지속 동기, 회복 링크 제공
- 핵심 컴포넌트: 현재/최고 기록(절제 타이머에서 **파생** — 별도 하드코딩 카드 아님, PRD §0.6.4), 회복 링크 / 주요 CTA: 기록 보기 / 보조 CTA: 회복으로
- 표시 데이터: 현재 스트릭(= 타이머 일수), 최고 스트릭(아카이브된 과거 런 최댓값)
- 금지 카피: "리셋되면 끝"
- 수락 기준: 리셋 시에도 회복 경로 노출, 현재 값은 절제 타이머와 **단일 소스**(홈에 모순되는 중복 스트릭 카드 금지, PRD §0.6.4)
- Mobbin 수집: `05_Streak`

### 매일 1분 체크인 (Daily Check-in)
- 우선순위: P0 / 사용자 상태: 어색함
- 목적: 감정·트리거·난이도 기록 (복기 다이어리의 입력 소스)
- 핵심 컴포넌트: mood(enum), trigger_type(편집 가능 태그 세트 §0.6.7 재사용), urge_level(1~5) / 주요 CTA: 완료 / 보조 CTA: 나중에
- 표시 데이터: enum 선택지 + 편집 가능 트리거 태그
- 금지 카피: "왜 또", 질문 과다
- 수락 기준: 1분 이내 완료, 완료율 가설 40%, **규율 상태 점검은 별도 3-tap 모델을 만들지 않고 단일 Discipline 3-state(§0.6.2)로 라우팅**(중복 상태 경로 금지 — PRD §0.6 / 메모 §9), mood/trigger/urge는 복기 흐름의 입력으로만 사용
- Mobbin 수집: `12_Checkin_Journal`

### 위기 버튼 (Urge Button)
- 우선순위: P0 / 사용자 상태: 급함
- 목적: 즉시 대응, 충동 지연
- 핵심 컴포넌트: 큰 버튼, 타이머 / 주요 CTA: 5분 버티기 / 보조 CTA: 대체 활동
- 표시 데이터: 타이머
- 금지 카피: "참아야 한다"
- 수락 기준: 1탭 진입, 사용 후 이탈 방지 가설 30%
- Mobbin 수집: `13_Crisis`

### 5분 대체 활동 (5-minute Alternative Activity)
- 우선순위: P0 / 사용자 상태: 충동 지연 중
- 목적: 대체 행동으로 시간 지연
- 핵심 컴포넌트: 활동 선택(enum), 타이머, 완료 / 주요 CTA: 활동 시작 / 보조 CTA: 다른 활동
- 표시 데이터: activity_type(enum)
- 금지 카피: 훈계형
- 수락 기준: 완료율 가설 35%
- Mobbin 수집: `13_Crisis`, `04_Habit`

### 복기 다이어리 / 다시 시작 (Reflection Diary · Relapse/Reset Recovery)
- 우선순위: P0 / 사용자 상태: 수치심 → 회복
- 목적: 실패/위기를 비난 없이 복기하고 다음 행동으로 잇는다. 기존 회복 화면을 **복기 다이어리로 재용도화**(PRD §0.6.6). 재방문 유도.
- 두 가지 진입(PRD §0.6.3):
  - **재발(타이머 리셋):** `무너졌어요 / 다시 시작` → 타이머 0으로 리셋 + 직전 런 아카이브 + **필수 복기**
  - **규율 슬립(`못 지켰어요`):** 타이머 리셋 없이 **가벼운(선택) 복기**로 진입
- 핵심 컴포넌트: 3-프롬프트 복기 폼(PRD §0.6.6) — ① **왜 그랬을까요?**(why, 짧은 자유 텍스트) ② **무엇이 계기였어요?**(트리거, 편집 가능 태그 다중선택 §0.6.7 + 자유 텍스트) ③ **다음엔 어떻게 해볼까요?**(다음 행동) / 주요 CTA: 다시 시작 / 보조 CTA: 복기 저장, 회복 루틴 보기
- 표시 데이터: 그 날 DayRecord(why/triggers/reflection/nextAction)에 기록(비공개 기본), 작성 시 배지 획득(복기 완료 / 다음 행동 작성 완료 / 회복 루틴 완료)
- 금지 카피: "실패자", "또 실패했네요", "위반", "벌점", 리셋 광고, 재발 직후 즉시 과금(§0.5.4 상속)
- 수락 기준: 재발은 타이머를 리셋하고 규율 슬립은 리셋하지 않음(§0.6.3 정확), 저마찰(≤3 프롬프트), 24h 재방문 가설 40%, 비난 0, 작성한 항목만큼 배지 부여
- Mobbin 수집: `12_Checkin_Journal`, `13_Crisis`

### 캐릭터 홈 (Character Home)
- 우선순위: P0 / 사용자 상태: 호기심
- 목적: 성장 몰입, 처벌 없음
- 핵심 컴포넌트: 캐릭터, 상태, XP, 절제 공간 / 주요 CTA: 돌보기 / 보조 CTA: 홈으로
- 표시 데이터: character_stage, XP
- 금지 카피: "벌받음", 캐릭터 비난
- 수락 기준: 처벌 없음, 동료/수호자 톤
- Mobbin 수집: `06_Character`

### 캐릭터 보상 (Character Reward)
- 우선순위: P0 / 사용자 상태: 성취
- 목적: 행동 보상 수령
- 핵심 컴포넌트: 보상(XP/배지/스킨) / 주요 CTA: 받기 / 보조 CTA: 닫기
- 표시 데이터: 보상 내역
- 금지 카피: "한 번 더 뽑기", 과금 압박
- 수락 기준: 보상=행동 결과, 수령률 가설 50%
- Mobbin 수집: `07_Rewards`

### 알림 권한 (Notification Permission)
- 우선순위: P0 / 사용자 상태: 설정
- 목적: 재방문 트리거 동의
- 핵심 컴포넌트: 권한 이유 설명 / 주요 CTA: 허용 / 보조 CTA: 나중에
- 표시 데이터: 없음
- 금지 카피: "필수 허용", 압박
- 수락 기준: 이유 먼저 설명, 허용률 가설 45%
- Mobbin 수집: `02_Permissions`

### 기본 설정 (Basic Settings)
- 우선순위: P0 / 사용자 상태: 관리
- 목적: 계정·알림·개인정보 관리 진입
- 핵심 컴포넌트: 알림, 개인정보(항목별 동의), 데이터 삭제 / 주요 CTA: 저장 / 보조 CTA: 없음
- 표시 데이터: 설정값
- 금지 카피: "자동 공유"

## 7. 화면별 수락 기준 (Screen-level acceptance criteria)

각 화면은 다음을 모두 만족해야 디자인/구현 단계로 넘어간다.
- 목적·사용자 상태·CTA가 위 표와 일치
- 금지 카피 0건 (COPY_POLICY 교차 검수)
- UX_RULES 위반 0건 (수치심·감시·완벽차단·자동공유 등)
- 우선순위 단계 준수 (P1/P2 기능을 P0 화면에 끌어오지 않음)
- 민감 데이터는 기본 비공개, enum/bucket 측정 (PRD §18)

## 8. 화면별 금지 카피 (Screen-level forbidden copy)

전 화면 공통 금지: 죄/타락/실패자/중독자, 치료·완치 단정, "완벽 차단/절대 우회 불가", "부끄럽지 않나요", "꼴찌", 성인 사이트명/URL/검색어, 과금 압박("무료는 부족", "한 번 더 뽑기"). 상세는 COPY_POLICY.md.
