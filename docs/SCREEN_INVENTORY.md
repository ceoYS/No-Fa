# SCREEN_INVENTORY.md

Korean NoFap 앱 — 화면 목록 (디자인·구현 전 범위 고정)

상위 문서: PRD §7, §8.1, §13, §14 / 같은 라운드: UX_RULES.md, COPY_POLICY.md
이 문서는 화면 **범위**를 고정한다. 화면 디자인·와이어프레임 생성은 하지 않는다.

---

## 1. 화면 목록 목적 (Screen inventory purpose)

- 디자인·프로토타입·구현 전에 MVP 화면 범위를 고정한다.
- 화면별 목적·CTA·금지 카피·수락 기준을 명시해 AI 임의 해석을 막는다.
- P0 / P0.5 / P1 / P2 경계를 명확히 한다. **새 화면 추가 금지** (PRD 변경 요청 선행).
- **단계 어휘는 운영 문서 어휘다.** 단계 어휘(`P0`, `P1`, `P2`, `MVP`, `프로토타입`, `보기 전용`)는 본 인벤토리·UX_RULES·COPY_POLICY 안에서만 쓰며, **사용자 UI 카피·placeholder·빈 상태로 노출하지 않는다** (PRD §0.5.8.1). 미래 기능이 P0에서 아직 없다면 placeholder를 만들지 않거나, 일반 어휘("곧 열려요")로만 표현한다.

화면 정의 항목: 화면명 / 우선순위 / 목적 / 사용자 상태 / 핵심 컴포넌트 / 주요 CTA / 보조 CTA / 표시 데이터 / 금지 카피 / 수락 기준 / Mobbin 벤치마크 수집.

---

## 2. P0 화면 (P0 screens — MVP Core Loop)

### Splash
- 우선순위: P0 / 사용자 상태: 첫 진입
- 목적: 브랜드 첫인상, 무겁지 않은 톤
- 핵심 컴포넌트: 로고, 짧은 문구 / 주요 CTA: 시작하기 / 보조 CTA: 없음
- 표시 데이터: 없음
- 금지 카피: "중독 치료", 치료/완치 단정
- 수락 기준: 무겁지 않은 톤, 3초 내 다음 진입 가능
- Mobbin 수집: `01_Onboarding`

### 온보딩 (Onboarding)
- 우선순위: P0 / 사용자 상태: 기대·민망함
- 목적: 문제·목표 파악, 수치심 최소화
- 핵심 컴포넌트: 3~5개 질문(enum 선택), "내가 무너지는 경로" 선택 / 주요 CTA: 다음 / 보조 CTA: 건너뛰기
- 표시 데이터: 선택지(enum)
- 금지 카피: "얼마나 심각한가요", 성적 고백 강요
- 수락 기준: 완료율 가설 70%, 자유서술 비강제
- Mobbin 수집: `01_Onboarding`

### 목표 설정 (Goal Setup)
- 우선순위: P0 / 사용자 상태: 결심
- 목적: 현실적 목표 설정
- 핵심 컴포넌트: 기간(7/14/30/커스텀), 이유, 알림 설정 / 주요 CTA: 목표 만들기 / 보조 CTA: 나중에
- 표시 데이터: goal_days bucket
- 금지 카피: "실패 금지", 압박형
- 수락 기준: 설정률 가설 80%
- Mobbin 수집: `01_Onboarding`, `04_Habit`

### 홈 대시보드 (Home Dashboard)
- 우선순위: P0 / 사용자 상태: 일상 진입
- 목적: 오늘 상태 요약, 명확한 오늘 행동
- 핵심 컴포넌트:
  - 헤더(아이브로우 "오늘도 지키는 중이에요" + 타이틀 "절제 시간")
  - **절제 타이머 카드** (일 수 + 시·분·초 카드형 표시, 헤더 H1 길이 폭주 방지, PRD §0.5.10)
  - 펫 스테이지(EmberCat — 보조 위치)
  - 절제 기록 카드 (현재/최장)
  - 오늘 보호한 자극 카드 (2칸 보호 메트릭 그리드)
  - **나의 규율 요약 카드** (요약 한 줄 + 보조 한 줄 + 최대 2개 컴팩트 규율 row + "규율 보기" CTA, 전체 리스트는 DisciplineScreen에서, PRD §0.5.9)
  - **오늘의 방 온기 카드** (상태 어휘만: 약함 / 잔잔함 / 안정 / 따뜻함, 숫자·점수 비표시, PRD §0.5.10)
  - 최근 7일 미니 스트립 (각 점 탭 가능 — 한 줄 요약, PRD §0.5.3.1)
- 주요 CTA: 오늘 상태 기록하기 / 보조 CTA: 충동 멈추기, 규율 보기, 전체 7일 보기
- 표시 데이터: 절제 경과 시간(일·시:분:초), 현재/최장 스트릭, character_stage, 오늘 체크인 여부, 오늘 보호한 자극 count(이미지/영상 mock count), discipline 상태(5-state: 지켰어요 / 확인 필요 / 흔들렸어요 / 회복 완료 / 오늘은 해당 없음 — PRD §0.5.9.1), room_warmth 상태(약함 / 잔잔함 / 안정 / 따뜻함 — 내부 계산값은 비공개), 최근 7일 점 상태(지킨 날 / 회복한 날 / 확인 필요 / 기록 전, PRD §0.5.3.2)
- 금지 카피: "참아라", 명령형, **내부 단계 어휘(P0/P1/P2/MVP/프로토타입/보기 전용)**, 운영 어휘(예: "중립 표기·처벌 없음", "코스메틱·케어 기반 보상", "결제·뽑기 없음", "enum", "정책"), "규율 위반"·"실패"·"벌점"·"포기" 류 1차 카피, 방 온기/규율의 **숫자·점수·퍼센트 노출 금지** (§0.5.10), 점수 부재 강조("깎이는 점수는 없어요") 금지, **"나만 보는 기록"·"비공개 기록"을 핵심 메트릭의 반복 배지로 노출 금지** (기록 비공개는 기본값 — UX_RULES §10.1 / COPY_POLICY §10.1), 메타포 단독 라벨("이어온 날", "조용해진 날", "아직", "잔불 캘린더") 금지 (PRD §0.5.8.1)
- 수락 기준: 오늘 할 행동 CTA 명확, **자기통제/충동 보호/회복 앱으로 명료히 읽힘** (PRD §0.5.8), 잔불·펫은 보조 메타포로만 사용, 타이머가 H1을 폭주시키지 않음, 정량 모델은 상태 어휘로만 번역됨(§0.5.10 D), 비공개 라벨은 홈의 반복 배지로 쓰지 않고 가치 기반 라벨("절제 기록", "최근 기록", "오늘 보호한 자극", "나의 규율") 사용
- Mobbin 수집: `04_Habit`, `05_Streak`

### 스트릭 카드 (Streak Card)
- 우선순위: P0 / 사용자 상태: 동기 확인
- 목적: 지속 동기, 회복 링크 제공
- 핵심 컴포넌트: 현재/최고 기록, 회복 링크 / 주요 CTA: 기록 보기 / 보조 CTA: 회복으로
- 표시 데이터: 현재/최고 스트릭
- 금지 카피: "리셋되면 끝"
- 수락 기준: 리셋 시에도 회복 경로 노출
- Mobbin 수집: `05_Streak`

### 매일 1분 체크인 (Daily Check-in)
- 우선순위: P0 / 사용자 상태: 어색함
- 목적: 감정·트리거·난이도 기록
- 핵심 컴포넌트: mood(enum), trigger_type(enum), urge_level(1~5) / 주요 CTA: 완료 / 보조 CTA: 나중에
- 표시 데이터: enum 선택지
- 금지 카피: "왜 또", 질문 과다
- 수락 기준: 1분 이내 완료, 완료율 가설 40%
- Mobbin 수집: `12_Checkin_Journal`

### 위기 버튼 (Urge Button)
- 우선순위: P0 / 사용자 상태: 급함
- 목적: 즉시 대응, 충동 지연
- 핵심 컴포넌트: 큰 버튼, 타이머 / 주요 CTA: 5분 버티기 / 보조 CTA: 대체 활동
- 표시 데이터: 타이머
- 금지 카피: "참아야 한다"
- 수락 기준: 1탭 진입, 사용 후 이탈 방지 가설 30%
- Mobbin 수집: `13_Crisis`

### 5분 대체 활동 (5-minute Alternative Activity)
- 우선순위: P0 / 사용자 상태: 충동 지연 중
- 목적: 대체 행동으로 시간 지연
- 핵심 컴포넌트: 활동 선택(enum), 타이머, 완료 / 주요 CTA: 활동 시작 / 보조 CTA: 다른 활동
- 표시 데이터: activity_type(enum)
- 금지 카피: 훈계형
- 수락 기준: 완료율 가설 35%
- Mobbin 수집: `13_Crisis`, `04_Habit`

### 실패/리셋 후 회복 (Relapse/Reset Recovery)
- 우선순위: P0 / 사용자 상태: 수치심
- 목적: 회복 루프, 재방문 유도
- 핵심 컴포넌트: 원인(enum), 감정(enum), 다음 행동, 회복 플랜 / 주요 CTA: 다시 시작 / 보조 CTA: 회복 플랜 보기
- 표시 데이터: enum 기록(비공개 기본)
- 금지 카피: "실패자", "또 실패했네요"
- 수락 기준: 24h 재방문 가설 40%, 비난 0
- Mobbin 수집: `12_Checkin_Journal`, `13_Crisis`

### 캐릭터 홈 (Character Home)
- 우선순위: P0 / 사용자 상태: 호기심
- 목적: 성장 몰입, 처벌 없음
- 핵심 컴포넌트: 캐릭터, 상태, XP, 절제 공간 / 주요 CTA: 돌보기 / 보조 CTA: 홈으로
- 표시 데이터: character_stage, XP
- 금지 카피: "벌받음", 캐릭터 비난
- 수락 기준: 처벌 없음, 동료/수호자 톤
- Mobbin 수집: `06_Character`

### 캐릭터 보상 (Character Reward)
- 우선순위: P0 / 사용자 상태: 성취
- 목적: 행동 보상 수령
- 핵심 컴포넌트: 보상(XP/배지/스킨) / 주요 CTA: 받기 / 보조 CTA: 닫기
- 표시 데이터: 보상 내역
- 금지 카피: "한 번 더 뽑기", 과금 압박
- 수락 기준: 보상=행동 결과, 수령률 가설 50%
- Mobbin 수집: `07_Rewards`

### 알림 권한 (Notification Permission)
- 우선순위: P0 / 사용자 상태: 설정
- 목적: 재방문 트리거 동의
- 핵심 컴포넌트: 권한 이유 설명 / 주요 CTA: 허용 / 보조 CTA: 나중에
- 표시 데이터: 없음
- 금지 카피: "필수 허용", 압박
- 수락 기준: 이유 먼저 설명, 허용률 가설 45%
- Mobbin 수집: `02_Permissions`

### 기본 설정 (Basic Settings)
- 우선순위: P0 / 사용자 상태: 관리
- 목적: 계정·알림·개인정보 관리 진입
- 핵심 컴포넌트: 알림, 개인정보(항목별 동의), 데이터 삭제 / 주요 CTA: 저장 / 보조 CTA: 없음
- 표시 데이터: 설정값
- 금지 카피: "자동 공유"
- 수락 기준: 항목별 동의, 삭제 경로 노출
- Mobbin 수집: `16_Privacy`

### 나의 규율 (My Discipline)
- 우선순위: P0 (상태 표시 + 행별 부분 갱신 + 가벼운 로컬 추가/삭제) / 사용자 상태: 자기 점검
- 목적: 사용자가 정한 사적 규율을 비공개로 점검·갱신 (PRD §0.5.9)
- 핵심 컴포넌트:
  - 오늘의 요약 카드("3개 중 2개 지켰어요" + 방 온기 상태 어휘 — 숫자 비표시, PRD §0.5.10)
  - 규율 리스트 + **5-state 상태 칩** (지켰어요 / 확인 필요 / 흔들렸어요 / 회복 완료 / 오늘은 해당 없음 — PRD §0.5.9.1)
  - **행별 상태 변경 시트** (5-state 칩 선택) — `흔들렸어요` 선택 시 같은 카드 안에 회복 행동 제안 한 줄 + 회복 화면으로 보조 CTA
  - **회복 완료 자동 전이**: 회복 행동 완료 시 해당 규율은 `회복 완료`로 전이
  - 규율 다루는 방식 안내 (단계 어휘 노출 금지)
  - 헤더 "+규율 추가" 버튼, 행별 "삭제" 버튼
  - 추가 바텀시트(타이틀 "새 규율 추가" + 입력 + 카테고리 칩 + 추가하기/취소)
  - 삭제 확인 시트("이 규율을 삭제할까요?" / "삭제해도 언제든 다시 추가할 수 있어요.")
- 주요 CTA: 홈으로 돌아가기 / 보조 CTA: 규율 추가, 규율 삭제, 상태 변경, 회복 행동으로
- 표시 데이터: discipline_rule(로컬 in-memory 선택지 + 사용자 입력 라벨), discipline_category(밤 시간 / 충동 / 검색 / SNS·숏폼 / 수면), discipline_status(5-state, PRD §0.5.10 A 매핑), room_warmth 상태(내부 계산은 비공개)
- 확인 입력 동선(PRD §0.5.9.2 상속):
  1. 체크인 마지막 스텝의 "오늘의 규율 점검" 카드에서 일괄 점검 (`지켰어요` / `흔들렸어요` / `해당 없음` 3-tap 칩)
  2. 본 화면에서 행별 부분 갱신(5-state)
  3. 회복 행동 완료 시 자동 `회복 완료` 전이
- 금지 카피: "규율 위반", "실패", "벌점", "위반 횟수", "포기", "강등", 공개 약속/감시 프레이밍, 펫 처벌 연결, 정량 점수·등급·랭킹 노출(§0.5.10), 점수 부재 강조 카피("깎이는 점수는 없어요" 류 — 점수 프레임 환기 회피, PRD §0.5.8.1), **내부 단계 어휘(`P0`, `P1`, `프로토타입`, `보기 전용`) 노출 금지**, "다음 단계에서 열어요 (P1)" 류 단계 라벨 금지 — 필요 시 "곧 열려요" 같은 일반 어휘로 표현, "선택지로만 기록" (`enum` 노출 금지), 삭제 카피 "이 규율을 포기할까요?" 금지
- 수락 기준: 비공개·비처벌·비감시·회복 우선, P0 로컬 추가/삭제 및 상태 갱신은 in-memory(영속화·서버·인증·DB 없음), Home 요약과 같은 state 공유, 거친 실패 상태 없음, 정량 모델은 상태 어휘로만 번역됨, 비공개 라벨을 헤더 배지로 노출하지 않음(UX_RULES §10.1), 사용자 UI에 단계 어휘·운영 어휘 0건
- Mobbin 수집: `04_Habit`, `12_Checkin_Journal`

### 최근 기록 (Recent Records · Calendar Tab)
- 우선순위: P0 / 사용자 상태: 회고
- 목적: 최근 7일의 자기통제 상태를 한 화면에서 한눈에. 가짜 월 placeholder 없이 7일을 1차 표면으로(PRD §0.5.3.1).
- 핵심 컴포넌트:
  - 화면 타이틀: `최근 기록` (또는 `절제 기록`) — `잔불 캘린더` 메타포 단독 타이틀 금지
  - 최근 7일 카드 1장 (요일 라벨 + 점 + 오늘 강조)
  - 각 점 **탭 가능** → 하단/시트로 그 날의 한 줄 요약 표시 (예: "5월 24일 — 지킨 날 · 체크인 완료, 흔들림 없음")
  - 범례: `지킨 날` / `회복한 날` / `확인 필요` / `기록 전` (메타포 단독 라벨 금지)
- 주요 CTA: 홈으로 돌아가기 / 보조 CTA: 오늘 체크인 진입
- 표시 데이터: 최근 7일 점 상태(kept / recovered / needs_check / untracked, PRD §0.5.3.2), 각 날의 한 줄 요약(체크인 여부 + 흔들림 수 + 회복 행동 수, 자유서술 0)
- 금지 카피: `이번 달 (보기 전용)` 섹션 자체 금지, `월/주 전체 보기는 P1에서 열려요` 류 단계 placeholder 금지, 메타포 단독 라벨(`이어온 날`, `조용해진 날`, `아직`) 금지, 빨강/이진 톤 금지, 점수·퍼센트·등급 비표시, 자유서술 첨부·사진 첨부 비제공
- 수락 기준: 최근 7일 점이 모두 탭 가능하며 탭 시 한 줄 요약 표시, 가짜 월 placeholder 0건, 단계 어휘 0건, 사용자 라벨이 §0.5.3.2 enum과 일치, 톤은 잔불·조명·온도의 연속 스펙트럼만(빨강·이진 금지), 미래의 월/주 전체 보기·날짜 상세 페이지는 본 화면에 노출하지 않음(필요 시 도움말에 한 줄 안내)
- Mobbin 수집: `05_Streak`, `14_Stats`
