상태: 제품 구조 적응 문서 / 기능 구현 아님 / 이미지·에셋 생성 아님 / 최종 색·캐릭터·UI 확정 아님 (사람 승인 전)

> **이 문서의 역할**
> - 이전에 탐색한 Failendar(실패 달력 앱) 컨셉에서 **유용한 제품 구조만 추출**해 NoF 언어로 변환한다.
> - Failendar를 그대로 복제하지 않는다. 실패-우선 프레이밍·빨강/초록 이진·실패 레벨 판정·리셋 광고는 전부 reject한다.
> - 기능·코드·이미지는 만들지 않는다. SCREEN_INVENTORY·UX_RULES·COPY_POLICY·MONETIZATION 경계를 그대로 상속한다.
> - 화면 추가 최소화 원칙은 유지하되, PRD §0.5.3.1 확장 패치로 P0에 **`최근 기록` 탭 1개**가 추가됐다(7일 카드 + 점 탭 한 줄 요약까지). 그 외 Failendar 구조는 기존 P0/P0.5 화면에 흡수한다(§4).

## 1. Executive decision (요지 결정)

- **채택:** Failendar의 **구조적 골격**(달력·타이머·키워드 체크·알림·통계)은 자기통제 앱에서 검증된 유용한 뼈대다. 이 골격만 가져온다.
- **변환:** 모든 골격을 NoF의 **회복·펫·비감시** 언어로 다시 쓴다. "실패 달력"이 아니라 "회복(잔불) 달력", "성공 타이머"가 아니라 "절제 일수(잔불 스트릭)", "성공/실패 키워드"가 아니라 "트리거 키워드 + 보호 행동 체크리스트"다.
- **거부:** Failendar의 **정서적 코어**(실패-우선·수치·이진 판정·리셋 즉시 과금)는 NoF 포지션(`DIRECT_COMPETITOR_VISUAL_AUDIT §5`, `DESIGN_RESET_BRIEF §8`)과 정면 충돌한다. 0건으로 유지한다.
- **펫이 코어 정서 엔진이다.** Failendar는 달력이 코어였지만, NoF는 **펫(고양이) 돌봄**이 매일 여는 이유다. 달력·타이머·통계는 펫 세계관(Ink & Ember 잔불)을 **떠받치는 데이터 층**이지 그 자체가 주인공이 아니다.
- **MVP는 과설계 금지.** 달력·통계의 풍부한 기능은 P1+로 미룬다. P0는 "오늘 흔들림을 가볍게 기록 → 펫의 잔불이 이어진다"는 최소 루프만.

한 줄 검수(UX_RULES §1): "이 구조가 사용자를 깎아내리는가, 다시 일어서게 하는가?"

## 2. Borrow / Convert / Reject 표

| Failendar 원형 | 처리 | NoF 변환 결과 | 근거 |
|---|---|---|---|
| 실패 달력 (failure calendar) | Convert | **회복 캘린더 (Recovery Calendar)** — 사용자 화면 1차 타이틀은 `최근 기록` 또는 `절제 기록`. ("잔불 캘린더"는 내부 메타포 명칭이며 화면 단독 타이틀로 쓰지 않는다 — PRD §0.5.8.1) | UX_RULES §3·§10.7, PRD §0.5.3 |
| 실패 기록 (failure record) | Convert | **흔들림 기록 / Urge Log / Recovery Log** — 충동을 사건으로, 비공개 기본 | AUDIT §3-3, UX_RULES §4 |
| 성공 타이머 (success timer) | Convert | **절제 일수 / 잔불 스트릭 (Ember streak)** — 현재/최고 중립 표기 | SCREEN_INVENTORY 스트릭 카드 |
| 성공/실패 키워드 (success/fail keywords) | Convert | **트리거 키워드(enum) + 보호 행동 체크리스트** — 원터치 체크, 자유서술 강요 0 | COPY_POLICY §6, UX_RULES §4 |
| 루틴 알람 (routine alarm) | Convert | **위험 시간대 보호 리마인더** — 사용자가 정한 리스크 타임 | UX_RULES §6 위기, 알림 권한 화면 |
| 통계 (statistics) | Convert | **패턴 인사이트 / 회복 리포트** — 시간대 트리거·회복률·스트릭 히스토리 | AUDIT §3, P1 기본 통계 |
| 데이터 보존 (data retention) | Convert | **장기 회복 아카이브 (프리미엄 패턴 히스토리)** — 비랜덤·비능력 P2 | MONETIZATION, PET_CUSTOMIZATION §9 |
| 캘린더 월/주 뷰 구조 | Borrow | 월/주 뷰 + 날짜별 기록 + 빠른 기록 버튼 + 패턴 가시성 | — |
| 타이머 구조 | Borrow | 스트릭 일수 + 마일스톤 + 히스토리 | CHARACTER_SYSTEM_SPEC §4 성장 입력 |
| 키워드 구조 | Borrow | 원터치 체크 + 트리거 카테고리 + 보호 행동 | — |
| 알림 구조 | Borrow | 사용자 정의 루틴 리마인더 + 위험 시간 리마인더 | — |
| 통계 구조 | Borrow | 요일/시간 트리거 패턴 + 회복률 + 스트릭 히스토리 + 주간/월간 인사이트 카드 | — |
| 실패-우선 제품 프레이밍 | **Reject** | 회복-우선으로 대체. "오늘 실패했나요?" 류 진입 0 | RESET_BRIEF §8, COPY_POLICY §4 |
| 빨강 실패일 / 초록 성공일 이진 | **Reject** | 빨강·이진 금지. 잔불 톤의 연속 스펙트럼(밝음↔조용함) | RESET_BRIEF §8 |
| "오늘 실패했나요?" 류 카피 | **Reject** | "오늘 어땠어요?" / "1분 체크인" 담담한 톤 | COPY_POLICY §6 |
| 실패 심각도 Level 1/2/3 판정 | **Reject** | 판정 메카닉 금지. 트리거 enum은 분류이지 등급이 아님 | UX_RULES §4 |
| 리셋 광고 | **Reject** | 리셋 시점 광고 0 | MONETIZATION |
| 재발/리셋 직후 즉시 과금 | **Reject** | 회복 동선만 노출. 과금 압박 0 | UX_RULES §11, COPY_POLICY §11 |
| 민감 맥락 사진 첨부 | **Reject** | 사진 첨부 미도입. 측정은 enum/bucket/count | UX_RULES §10 |
| 수치·처벌·죄책감·종교/파트너 감시 톤 | **Reject** | 전부 배제. 비감시·기본 비공개 | RESET_BRIEF §9, AUDIT §4 |
| 펫이 굶주리거나 병들거나 처벌받는 연출 | **Reject** | 금지 상태(PET_CUSTOMIZATION §5). 펫은 조용해질 뿐 고통받지 않음 | CHARACTER_SYSTEM_SPEC §6 |

## 3. NoF product loop after adaptation (적응 후 제품 루프)

Failendar는 "실패를 달력에 칠하는" 루프였다. NoF는 **펫 돌봄을 코어로 둔 회복 루프**다.

```
[매일 진입]
  → 홈: 펫 + 잔불 상태 + 오늘 흔들림(최근 7일 점) 한눈에
  → 1분 체크인 (mood / trigger enum / urge level) — 담담히
  → 체크인 완료 시 펫에게 작은 간식/잔불 unlock (행동 보상)

[충동이 올 때]
  → 잠깐 멈춤(위기 버튼) → 호흡/타이머/대체활동
  → 버틴 것 = 흔들림 기록의 "보호 행동"으로 기록 → 성취로 인정(깎지 않음)

[위험 시간대]
  → 보호 리마인더(루틴 알람 변환) → "잠깐 멈춤" 진입 유도

[흔들림/재발이 있어도]
  → 처벌 0. 펫은 단계 강등 없음 → 잠시 조용해질 수 있음(ember_low)
  → 회복 행동을 하면 다시 잔불이 켜짐 → 회복 루프로 연결

[주기적]
  → 회복 캘린더(`최근 기록` 탭)에서 패턴 가시화 → 패턴 인사이트(시간대·트리거·회복률)
  → "나를 이해하는" 프레이밍 (치료/완치 단정 아님)
```

핵심: 달력·타이머·통계는 **펫의 잔불(절제 공간 안정)을 설명하는 데이터**다. 사용자가 보는 1순위는 펫이고, 구조는 그 뒤를 받친다.

## 4. P0 화면 영향 (P0 screen impact)

Failendar 구조는 **기존 P0 화면에 흡수**하되, PRD §0.5.3.1 확장 패치로 **`최근 기록` 탭 1개**가 추가된다(7일 카드 + 점 탭 한 줄 요약까지).

| P0 화면 | Failendar 구조 흡수 | 비고 |
|---|---|---|
| 홈 대시보드 | 최근 7일 미니 스트립 + 중립 스트릭. 각 점 **탭 가능** → 한 줄 요약 | 월 뷰 placeholder 금지. 단계 어휘 노출 0 |
| **최근 기록 (Calendar 탭)** | 7일 카드 1장 + 점 탭 시 한 줄 요약 | PRD §0.5.3.1·SCREEN_INVENTORY §2. 화면 1차 타이틀 `최근 기록`/`절제 기록`. `잔불 캘린더` 금지 |
| 스트릭 카드 | 성공 타이머 → **절제 일수(현재/최고) + 마일스톤** | 리셋 시 회복 링크 유지 |
| 매일 1분 체크인 | 키워드 구조 → trigger_type 선택지 + urge_level + (선택) 보호 행동 체크 + **마지막 스텝 "오늘의 규율 점검" (5-state 3-tap)** | 1분 이내, 자유서술 비강제, PRD §0.5.9.2 |
| 위기 버튼 / 5분 대체활동 | 위기 시 보호 행동이 흔들림 기록에 누적 | 위기 사용 = 성취 |
| 실패/리셋 후 회복 | 실패 기록 → 흔들림/회복 로그(원인·감정·다음 행동 선택지) | 24h 재방문, 비난 0, 회복 완료 시 규율 자동 `회복 완료` 전이 |
| 나의 규율 | 5-state 상태 + 행별 부분 갱신 + 가벼운 로컬 추가/삭제 | PRD §0.5.9.1·§0.5.9.2 |
| 캐릭터 홈 | 절제 일수·체크인·회복 행동이 **펫 절제 공간 안정**으로 시각화 | 강등 없음. 펫·잔불은 보상 층 |
| 알림 권한 / 기본 설정 | 루틴 알람 → 위험 시간대 보호 리마인더 설정 진입 | 이유 먼저 설명 |

**P1로 미루는 것(MVP 과설계 금지):**
- 캘린더 월/주 전체 뷰 + 날짜별 상세 페이지
- 패턴 인사이트/회복 리포트 화면 (기본 통계는 P1)
- 장기 회복 아카이브(프리미엄 히스토리, P2)
- 위험 시간대 다중 리마인더 세밀 설정
- 규율 영속 편집 UI / 템플릿 / 위험 시간 규율 리마인더 / 규율별 인사이트
- 규율 자동 시간 기반 리셋

**사용자 UI에 단계 어휘(`P0`, `P1`, `프로토타입`, `보기 전용`) 노출 금지** (PRD §0.5.8.1). 미래 기능 placeholder는 만들지 않는다(UX_RULES §10.8).

P0는 **홈 + 1분 체크인 + 스트릭 + 회복 + 펫 + 나의 규율 + 최근 기록 탭**의 최소 루프만 검증한다(`PREBUILD_DEMAND_VALIDATION_PLAN` 게이트 전제).

## 5. 펫 시스템 영향 (Pet system impact)

펫은 Failendar 구조의 **정서적 번역기**다. 모든 데이터 변화는 펫 상태/잔불로 부드럽게 표현된다(PET_CUSTOMIZATION_SPEC §5·§6 상속).

- **캘린더 점 → 펫 절제 공간 톤.** 기록이 쌓일수록 펫 곁 잔불·공간 안정도가 올라간다. 빨강 실패일/초록 성공일 이진 금지 → 잔불의 밝음↔조용함 연속 톤.
- **스트릭/타이머 → 성장 입력.** 스트릭 단독이 아니라 체크인 누적·위기 버틴 횟수·회복 루프 완료를 함께 본다(CHARACTER_SYSTEM_SPEC §4). 스트릭이 끊겨도 단계 강등 없음(§5).
- **흔들림/재발 → `taking_a_break` / `ember_low`.** 펫은 굶거나 병들거나 처벌받지 않는다. 잠시 조용해지거나 잔불이 약해질 뿐이고, **회복 행동을 하면 다시 잔불이 켜진다.**
- **보호 행동/체크인 → 케어 기반 보상.** 음식·간식·따뜻한 빛·방 아이템·장난감·잠자리·잔잔한 애니메이션. 능력치 상승·확률형 뽑기 0(PET_CUSTOMIZATION §8).
- **위험 시간 리마인더 → 펫의 "곁에 머무는" 존재감.** 감시·훈계 톤 금지. "잠깐 멈춤" 동료 톤.

펫 가드레일(절대): 굶주림·병듦·죽음·울음·퇴화·처벌 연출 0. 펫이 사용자의 흔들림으로 고통받는 표현 금지(PET_CUSTOMIZATION §5 금지 상태).

## 6. 캘린더 / 타이머 / 키워드 / 알림 / 인사이트 매핑

### 6.1 캘린더 (Calendar)
- 차용: 월/주 뷰, 날짜별 기록, 빠른 기록 버튼, 패턴 가시성.
- 변환: **회복 캘린더**. 사용자 화면 1차 타이틀은 `최근 기록` 또는 `절제 기록`(메타포 단독 타이틀 금지, PRD §0.5.8.1). 날짜 점은 `지킨 날` / `회복한 날` / `확인 필요` / `기록 전` 4상태 enum으로(PRD §0.5.3.2), 톤은 잔불·조명·온도의 연속 스펙트럼만. 빨강/이진 금지.
- P0: 홈 미니 7일 스트립 + `최근 기록` 탭(7일 카드 + 점 탭 한 줄 요약). 월/주 전체 뷰·날짜 상세 페이지는 P1. **사용자 UI에 "월/주 전체 보기는 P1에서 열려요" 류 단계 placeholder 카피로 노출하지 않는다.**

### 6.2 타이머 (Timer)
- 차용: 스트릭 일수, 마일스톤, 히스토리.
- 변환: **절제 일수 / 잔불 스트릭**. 현재/최고 중립 표기. 초 단위 카운트다운 공포 연출 금지(CLAUDE_DESIGN_PATTERN_BRIEF §7). 마일스톤 = 펫 절제 공간 안정 단계.
- 리셋: 즉시 0 가혹 시각화 금지. 누적 여정·펫 유지(UX_RULES §3).

### 6.3 키워드 (Keyword)
- 차용: 원터치 체크, 트리거 카테고리, 보호 행동.
- 변환: **트리거 키워드 enum**(지루함/스트레스/외로움/밤/특정 앱 맥락 등 — 사이트명·노골 표현 0) + **보호 행동 체크리스트**(잠깐 멈춤/대체활동/환경 차단/누군가에게 말하기 등).
- 규칙: 분류이지 심각도 등급(Level 1/2/3)이 아니다. 자유서술 강요 0, 비공개 기본.

### 6.4 알림 (Alarm)
- 차용: 사용자 정의 루틴 리마인더, 위험 시간 리마인더.
- 변환: **위험 시간대 보호 리마인더**. 사용자가 정한 리스크 타임에 "잠깐 멈춤" 동료 톤으로. "참아라" 명령형·죄책감 알림 금지.
- 권한: 이유 먼저 설명, 압박 0(SCREEN_INVENTORY 알림 권한).

### 6.5 인사이트 (Insights)
- 차용: 요일/시간 트리거 패턴, 회복률, 스트릭 히스토리, 주간/월간 인사이트 카드.
- 변환: **패턴 인사이트 / 회복 리포트**. "나를 이해하는" 자기학습 프레이밍. "Brain Rewiring %"·"치료/완치" 의사과학 정량화 금지(COPY_POLICY §4, AUDIT §4).
- 단계: 기본 통계 P1, 장기 아카이브 P2(비랜덤·비능력).

## 7. Monetization implications (수익화 함의)

MONETIZATION 경계(PET_CUSTOMIZATION_SPEC §9, UX_RULES §11) 상속. Failendar의 수익화 코어는 거부한다.

**허용 (P2, 검증 후, 비랜덤·비능력·성과 무관):**
- 장기 회복 아카이브 / 프리미엄 패턴 히스토리 (더 긴 기간·더 세밀한 인사이트 "선택")
- 비랜덤 코스메틱: 펫 스킨/눈·털 색 프리셋, 방 테마, 조명, 집중 사운드
- 더 세밀한 패턴 분석(프로) — "더 깊게 이해하고 싶은 사용자의 선택"으로만

**금지:**
- 리셋 광고 / 재발·리셋 직후 즉시 과금 (Failendar 수익화 코어 — 전면 거부)
- 스트릭 복구권 / 흔들림(실패) 삭제권 / 차단 해제권 판매
- 확률형 뽑기, pay-to-win, 위기 지원·회복의 유료화
- "무료는 부족" 압박, 보상 화면 "한 번 더 뽑기"

핵심: **회복·위기·체크인·기본 캘린더는 영구 무료.** 유료는 "더 깊은 이해"와 "코스메틱"뿐이며, 흔들린 순간을 돈으로 건드리지 않는다.

## 8. Copy rules (카피 규칙)

COPY_POLICY 상속 + Failendar 변환 전용 규칙.

**쓴다 (승인):**
- "오늘 어땠어요?" / "1분 체크인" (실패 질문 아님)
- 캘린더 라벨: `지킨 날` / `회복한 날` / `확인 필요` / `기록 전` (PRD §0.5.3.2)
- 화면 1차 타이틀: `최근 기록` 또는 `절제 기록`
- 디시플린 상태: `지켰어요` / `확인 필요` / `흔들렸어요` / `회복 완료` / `오늘은 해당 없음` (PRD §0.5.9.1)
- "흔들렸어도 괜찮아요. 회복 행동을 한 가지 해볼까요?"
- "지금은 선택을 늦추는 시간이에요." / "오늘도 함께 버텼어요."
- "이 시간대에 흔들림이 많았어요." (패턴 = 자기이해)

**금지:**
- "오늘 실패했나요?" / "또 실패했네요" / "실패일"
- 메타포 단독 라벨: `이어온 날`, `조용해진 날`, `아직`, `잔불 캘린더`(화면 1차 타이틀로)
- "잔불이 이어지고 있어요" 류 메타포 단독 1차 카피 (보조 메타포로만 사용)
- 빨강 실패일·초록 성공일 라벨, "Level 1/2/3 실패"
- 디시플린 평가·처벌 어휘: `위반`, `벌점`, `실패`(상태로), `포기`, `점수 차감`, `강등`
- 점수 부재 강조 카피("깎이는 점수는 없어요")
- 단계 어휘: `P0`, `P1`, `P2`, `MVP`, `프로토타입`, `보기 전용` (사용자 UI 노출 금지)
- 운영 어휘: `enum`, `bucket`, `정책`, `코스메틱`, `능력치` ("선택지로만 기록"으로 표현)
- "Brain Rewiring %" / "치료된다" / "완치"
- "리셋하고 다시 광고 보기" / 재발 직후 결제 유도 카피
- 수치·종교·남성성·감시·완벽차단 보장 어휘(COPY_POLICY §4)

용어 변환 사전:
- 실패 달력 → 회복 캘린더 (사용자 화면 타이틀은 `최근 기록`/`절제 기록`)
- 실패 기록 → 흔들림 기록 / 회복 기록
- 성공 타이머 → 절제 일수
- 성공/실패 키워드 → 트리거 선택지 + 보호 행동
- 루틴 알람 → 위험 시간대 보호 리마인더
- 통계 → 패턴 인사이트 / 회복 리포트
- 데이터 보존 → 장기 회복 아카이브

## 9. Safety / privacy guardrails (안전·개인정보 가드레일)

UX_RULES §10, RESET_BRIEF §9, PET_CUSTOMIZATION §5 상속.

- **사진 첨부 미도입.** 민감 맥락 사진/스크린샷 첨부 기능 만들지 않는다.
- **원문 미저장.** 사이트명/URL/검색어 원문 저장·표시 금지. 측정은 enum/bucket/count.
- **기본 비공개.** 흔들림 기록·트리거·체크인·캘린더는 기본 비공개. 자동 공유 0. 공유 범위는 사용자가 직접 선택(기본 OFF).
- **비감시.** 파트너가 활동 리포트를 받는 구조 금지. 동료성은 동의 기반·응원자 프레임.
- **자유서술 비강제.** 트리거·감정은 enum 중심. 성적 고백 강요 0. 자유서술은 선택·비공개.
- **펫 안전.** 펫은 굶주림·병듦·죽음·처벌 연출 0. 흔들림은 펫 단계 강등이 아니라 잠깐 조용해짐(회복 시 복귀).
- **삭제 경로.** 동의 철회·데이터 삭제 경로 명확히 제공.
- **정직 고지.** "완벽 차단/절대 우회 불가" 금지. 치료/완치 단정 금지.
