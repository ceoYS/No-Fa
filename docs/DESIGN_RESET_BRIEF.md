## 1. 왜 Baseline B("따뜻한 종이")를 reject하는가

Baseline B는 "따뜻한 종이"를 기본 비주얼 기준으로 채택했다. 시각적으로 reject한다. 이유:

1. **generic beige / wellness diary로 읽힌다.** 베이지 단색 무드는 명상·일기·습관 앱(Calm/Finch/Bloom/breeze 류)과 구분되지 않는다. DESIGN_DIRECTION §3·§4("베이지 일색 회피", "제네릭 파스텔/베이지 드리프트 금지")와도 정면으로 배치된다.
2. **카테고리 정체성이 비주얼에서 사라진다.** "NoF가 무슨 문제를 푸는 앱인지"가 화면에서 읽히지 않는다. 인접 wellness 앱 입력이 과도했고 직접 경쟁사 입력이 부족했던 결과다.
3. **차별화 좌표가 없다.** 직접 경쟁사 감사(`DIRECT_COMPETITOR_VISUAL_AUDIT.md` §5)는 빈 포지션이 "차분 프리미엄 + 따뜻한 인간미 + 비감시 자기통제"임을 보여준다. 베이지 워크북은 이 좌표를 점유하지 못한다.

폐기/보류 항목:

| 버릴 것 | 이유 |
|---|---|
| "따뜻한 종이"를 기본 비주얼 기준으로 채택 | generic beige / wellness diary로 읽힘. 카테고리 정체성 소실 |
| 인접 wellness/일기 앱 과다 영향 | 직접 경쟁사 차별화 입력 부족의 결과 |
| 베이지 단색 무드 보드 | DESIGN_DIRECTION §4와 배치. 톤 좌표 재설정 필요 |
| C안 "호박 등불"을 위기/차단 포인트로 흡수한다는 잠정 결정 | 톤 좌표를 다시 잡으므로 보류. 색 확정은 사람 승인 후 |

유지: P0 핵심 8화면 우선순위(홈/체크인/충동/대체활동/회복/캐릭터홈/차단/파트너공유)는 그대로 유효. 바뀌는 건 **톤·비주얼 좌표**뿐이다.

## 2. 직접 경쟁사에서 확인한 금지 톤

`DIRECT_COMPETITOR_VISUAL_AUDIT.md` §4 기준. 직접 경쟁사가 실제로 쓰지만 NoF가 따라 하면 안 되는 톤.

**카피/프레이밍 (anti-pattern)**
- ❌ "DON'T BREAK THAT PROMISE… FOR A FEW SECONDS OF PLEASURE" (QUITTR Panic) — 수치·죄책감 유도.
- ❌ "Side effects of Relapsing: REDUCED PERFORMANCE" — 공포·신체 위협 프레이밍.
- ❌ "Brain Rewiring %", "Recovery 0%", 회복 레벨 — 의사과학 정량화, 완치 단정 인접.
- ❌ 남성성/의지 프레이밍("전사", "의지로 이겨라", 슬랭) — 모욕·과도한 남성 코드.
- ❌ 노골적 단어 전면 노출 — NoF 기본 카피는 "절제/자기통제/유해 디지털 환경".
- ❌ "완벽 차단/우회 불가" 보장.

**UI/연출 (anti-pattern)**
- ❌ 빨강 경고색 + 하강/X 연출로 실패 강조 (QUITTR "Reset Counter" 빨강, "cigarettes over" 빨강).
- ❌ 스트릭 즉시 0 리셋의 가혹한 시각화.
- ❌ 게임 경제 과다 — 코인/재화/상점/XP뽑기/랭킹 강등.
- ❌ 소셜 프루프로 불안 유도 — "X명이 슬퍼함 😔" 식 부정 감정 카운트.
- ❌ 감시형 accountability — 파트너가 사용자 활동 리포트를 받는 구조 (Covenant Eyes 계열).
- ❌ 우주/네온 + 회전 구체의 과한 spectacle — 게임 스킨처럼 보임 (QUITTR).

**톤 종합**

| 톤 축 | 경쟁사 경향 | NoF 입장 |
|---|---|---|
| 종교적 | Covenant Eyes 등 강함 | **배제** |
| 의료적 | Fortify, "brain rewiring" 계열 | 차분한 데이터는 차용, **완치 단정 배제** |
| 남성성 과다 | Victory/Iron Will/MDF, QUITTR 슬랭 | **배제** |
| shame 유도 | QUITTR Panic/relapse | **배제** |
| 게임 spectacle | QUITTR 우주·네온 | 절제 미감으로 거름 |

## 3. NoF가 차지할 새 포지션

직접 경쟁사 지형의 빈 공간(`DIRECT_COMPETITOR_VISUAL_AUDIT.md` §5):

> **"수치·공포 없이 진지한, 한국 모바일 네이티브, 비감시 자기통제 앱."**

- 직접 경쟁사는 둘 중 하나다 — (a) 공포·수치·남성성(QUITTR/MDF 계열) 또는 (b) 종교·감시(Covenant Eyes 계열). **둘 다 아닌 차분한 자기통제** 포지션이 비어 있다.
- 거의 모든 직접 경쟁사가 영어권·서구 남성 코드. **한국어 톤·카카오 맥락·한국 UI 관습**이 무주공산.
- 동료성은 **기본 비공개 + 사용자가 공유 범위 결정**. 익명 슬랭 초대도, 감시 리포트도 아니다.
- 차단은 처벌이 아니라 **내가 정한 환경**. "완벽 차단" 군비경쟁에서 빠진다.
- 충동을 사건으로, 실패를 회복 루프의 시작으로. "rewiring %/치료" 단정 대신 **패턴 학습·자기이해** 프레이밍.

**새 톤 좌표:** 베이지 단색도, 우주 네온 spectacle도 아닌 **차분한 프리미엄 + 따뜻한 인간미 + 비감시 자기통제**. 따뜻함은 **색 온도가 아니라 카피·캐릭터·여백**으로 확보한다(beige 의존 금지).

## 4. 새 visual direction 후보 3개 (개요)

세 후보는 위 포지션을 서로 다른 무게중심으로 점유한다. 셋 다 §2의 금지 톤을 0으로 유지한다.

| 후보 | 한 줄 | 무게중심 | 베이스 |
|---|---|---|---|
| **A. Quiet Shield** | 조용한 보호막 / 프리미엄 다크 | 차분 프리미엄 ↑ | 다크/딥 뉴트럴 |
| **B. Clear Control** | 한국 네이티브 생산성·자기통제 앱 | 명료·자기주도 ↑ | 라이트 뉴트럴 + 액센트 |
| **C. Warm Companion** | 따뜻하지만 베이지 워크북 아닌 동료형 | 인간미·동료성 ↑ | 따뜻한 뉴트럴 (베이지 회귀 아님) |

## 5. 후보별 상세 (색감 · 타이포 · 홈 구조 · 캐릭터 톤 · 위기 버튼 톤)

각 항목은 **방향**이며 hex 확정이 아니다. 색은 사람 승인 후.

### A. Quiet Shield — 조용한 보호막 / 프리미엄 다크

- **색감:** 딥 잉크/차콜 베이스(검정 아님 — 음침 회피). 차분한 틸/세이지/뮤트 블루를 보조. 액센트는 따뜻한 골드 1점을 **소량**(spectacle 아닌 온기). 빨강 경고 없음. Opal식 "다크=프리미엄·집중"을 차용하되 네온·회전 spectacle은 배제.
- **타이포:** 한글 모던 산세리프(Pretendard 계열) 중심. 숫자(스트릭/타이머)는 또렷하되 카운트다운 공포 연출 없음. 위계는 여백으로, 굵기 대비는 절제.
- **홈 구조:** 어두운 캔버스 위에 캐릭터/수호 오브제가 차분히 빛나는 중앙 포커스. 오늘 상태 + 체크인 CTA가 1차. 스트릭은 중립 표기(현재/최고). 정보 과밀 회피.
- **캐릭터 톤:** 곁을 지키는 수호자. 은은한 발광, 정적인 존재감. 전투·능력치·울기·죽음 연출 금지.
- **위기 버튼 톤:** 어두운 배경에서 한 점의 차분한 빛으로 떠오르는 단일 진입점. "잠깐 멈춤" → 호흡/타이머/대체활동. 빨강·경고등·공포 카피 0.

### B. Clear Control — 한국 네이티브 생산성·자기통제 앱

- **색감:** 밝은 뉴트럴(오프화이트/소프트 그레이) 베이스 + 절제된 단일 액센트(차분한 블루/그린 계열). 한국 생산성·금융·습관 앱의 정돈된 미감. 베이지 일색 아님 — 뉴트럴은 그레이 톤이지 종이 톤이 아니다.
- **타이포:** 한글 가독 최우선. 명료한 위계, 카드/리스트 정렬, 데이터를 비난 없이 또렷하게(Fortify의 "차분한 데이터"만 차용, 임상 무게는 배제).
- **홈 구조:** 카드 기반 대시보드. 오늘 체크인 CTA가 최상단, 스트릭·캐릭터·차단 상태가 정돈된 카드로. "할 일 명확" 우선, 감정보다 통제감.
- **캐릭터 톤:** 조연으로서의 동료. 화면을 점유하기보다 상태를 알려주는 차분한 동반자. 게임화 과다 금지.
- **위기 버튼 톤:** 명확하고 접근성 높은 단일 버튼. 액센트 컬러로 또렷하되 경고색 아님. 1탭 진입, 짧은 카피, 타이머·대체활동 직결.

### C. Warm Companion — 따뜻하지만 베이지 워크북 아닌 동료형

- **색감:** 따뜻한 뉴트럴(웜 그레이/머스크/소프트 클레이) + 따뜻한 액센트 1~2점. **핵심: 베이지 단색 평면을 금지**하고, 채도·명도 대비와 깊이로 "종이 워크북"이 아닌 "따뜻한 공간"을 만든다. 따뜻함을 색이 아니라 깊이·여백·캐릭터로 확보.
- **타이포:** 부드러운 한글 산세리프. 둥근 모서리·여백 넉넉. 단, 유치·장난감 톤 회피(DESIGN_DIRECTION §3).
- **홈 구조:** 캐릭터와 "내 절제 공간"이 정서적 중심. 체크인 CTA는 캐릭터 곁의 자연스러운 행동으로 제시. 스트릭은 누적 여정으로(끊겨도 사라지지 않음을 시각화, UX_RULES §3).
- **캐릭터 톤:** 가장 캐릭터 중심. 따뜻한 동료, 곁에 머무는 존재. 실패 시 "쉬는 상태"(죽음·울기 아님). 성장은 단계적·따뜻하게.
- **위기 버튼 톤:** 따뜻하지만 분명한 진입점. "혼자 버티지 않아도 됩니다" 톤. 부드러운 색이되 흐릿하지 않게 — 급한 순간 인지 부하 최소.

## 6. 다크 베이스(후보 A) 장점과 위험

**장점**
- **차별화:** 직접 경쟁사 중 차분한 다크 프리미엄은 비어 있다. QUITTR 다크는 우주·네온 spectacle, Opal 다크는 quit-porn 카테고리가 아니다. NoF가 "진지한 다크 = 프리미엄·집중"을 선점할 수 있다.
- **베이지 드리프트 원천 차단:** 다크 베이스는 wellness diary로 오인될 여지가 구조적으로 낮다.
- **위기/저자극 적합:** 어두운 캔버스는 위기 화면의 "저자극·즉시 진입"과 캐릭터 발광 연출에 유리.
- **프리미엄 인상:** Opal이 입증한 "다크=고급" 효과.

**위험**
- **음침·무거움 리스크:** 다크가 잘못되면 "감시·경고·공포"로 읽힌다. → 검정 대신 딥 뉴트럴, 따뜻한 액센트로 온기 확보 필수.
- **scareware 인접:** 다크 + 빨강 = QUITTR Panic 미감. → **빨강 경고색 전면 금지**(§8).
- **따뜻함 손실:** DESIGN_DIRECTION의 "따뜻한 자기개선" 목표와 충돌 가능. → 따뜻함을 카피·캐릭터·여백으로 보강. 다크는 베이스일 뿐 톤 전체가 아니다.
- **한국 네이티브 정합성:** 한국 주류 앱은 라이트 베이스가 많음. 다크가 낯설게 느껴질 리스크. → 후보 B와 비교 검증 필요.

결론: 다크는 **유력 후보이나 단독 확정 금지**. 사람이 A/B/C를 비교한 뒤 결정.

## 7. beige / wellness diary drift 방지 규칙

Baseline B의 실패를 반복하지 않기 위한 가드레일. 모든 후보(특히 C)에 적용.

- 🚫 **베이지 단색 평면을 기본 캔버스로 쓰지 않는다.** 따뜻한 뉴트럴을 쓰더라도 깊이·대비·액센트로 차별화한다.
- 🚫 명상·일기·습관 앱(Calm/Finch/Bloom/breeze 류)의 무드 보드를 직접 차용하지 않는다. 인접 wellness 입력은 **보조**일 뿐 1순위가 아니다.
- 🚫 "종이 질감·워크북·다이어리" 메타포를 비주얼 중심으로 삼지 않는다.
- ✅ 화면을 보면 "자기통제/절제 환경을 지키는 앱"임이 읽혀야 한다. wellness journal과 구분돼야 한다.
- ✅ 따뜻함은 색 온도가 아니라 **카피·캐릭터·여백·동료성**으로 확보한다.
- ✅ 검수: "이 화면을 명상 앱 스크린샷 옆에 두면 구분되는가?" 아니면 drift.

## 8. QUITTR식 공포·수치·빨강 리셋 방지 규칙

- 🚫 **빨강 경고색**을 실패/위기/리셋에 쓰지 않는다. 하강·X·경고등 연출 금지.
- 🚫 **스트릭 즉시 0 리셋**을 가혹하게 시각화하지 않는다. 누적 데이터·캐릭터·여정은 유지됨을 보여준다(UX_RULES §3).
- 🚫 **공포·수치·약속배신·신체위협 카피** 0건 — "약속을 깨지 마", "성능 저하" 류 전면 금지(COPY_POLICY §4).
- 🚫 **의사과학 정량화** — "Brain Rewiring %", "Recovery %", "치료/완치" 단정 금지(COPY_POLICY §4).
- 🚫 **우주/네온/회전 구체 spectacle** — 게임 스킨화 금지.
- 🚫 **부정 감정 소셜 프루프** — "X명이 슬퍼함" 류 불안 유도 금지.
- ✅ 위기는 "지금은 선택을 늦추는 시간" 톤. 회복은 "여정이 끝난 건 아니에요" 톤.
- ✅ 위기 버튼 사용은 **성취로 인정**(XP). 사용했다고 깎지 않는다(UX_RULES §6).

## 9. Covenant류 감시·종교·도덕심판 방지 규칙

- 🚫 **감시형 accountability** 금지 — 파트너가 사용자 활동 리포트/로그를 받는 구조 금지(Covenant Eyes/Accountable2You/Ever Accountable 계열).
- 🚫 **종교적 미감·어휘** 금지 — "죄/타락", 신앙 프레이밍, 도덕 심판 톤(COPY_POLICY §4).
- 🚫 **자동 공유** 금지 — 실패 여부/원인은 기본 비공개. 공유 범위 기본 OFF(UX_RULES §8, §10).
- 🚫 "감시" 단어·감시자 프레임 금지.
- ✅ 동료성은 **동의 기반·기본 비공개·사용자가 범위 결정**. 응원자 프레임.
- ✅ 차단 로그·검색어·URL 원문 저장 금지. 측정은 enum/bucket/count(UX_RULES §10).
- ✅ 파트너 화면은 "혼자 버티지 않아도 됩니다 / 공유 범위는 직접 정할 수 있어요" 톤.

## 10. Claude Design에 줄 최종 프롬프트 초안

> 아래는 다음 디자인 라운드 핸드오프용 프롬프트 초안이다. 사람이 후보 A/B/C 중 방향을 고른 뒤, 고른 방향을 명시해 전달한다. 이 라운드 산출물은 **무드/방향 비교안 2~3개**이며 high-fidelity 화면은 사람 시각 승인 후.

```
역할: NoF(한국 모바일 네이티브 자기통제·절제 앱)의 visual direction을 잡는다.
이번 산출물은 무드/방향 비교안이다. 최종 화면·최종 색 확정이 아니다.

포지션:
- "수치·공포 없이 진지한, 한국 네이티브, 비감시 자기통제 앱."
- 톤 좌표: 차분한 프리미엄 + 따뜻한 인간미 + 비감시 자기통제.
- 따뜻함은 색 온도(베이지)가 아니라 카피·캐릭터·여백·동료성으로 확보한다.

후보 방향(아래 [사람이 고른 방향]으로 진행, 나머지는 참고):
A. Quiet Shield — 프리미엄 다크. 딥 뉴트럴 베이스 + 따뜻한 골드 소량 액센트.
   다크=프리미엄·집중(음침·공포 아님). 위기·캐릭터 발광에 유리.
B. Clear Control — 한국 네이티브 생산성·자기통제. 밝은 뉴트럴 + 단일 차분 액센트.
   카드 대시보드, 명료한 위계, 통제감 우선.
C. Warm Companion — 따뜻한 동료형(베이지 워크북 아님). 따뜻한 뉴트럴 + 깊이·대비.
   캐릭터·"내 절제 공간"이 정서 중심.

반드시 차별화(금지):
- QUITTR식: 빨강 경고·즉시 0 리셋·하강 연출, 공포/수치/약속배신/신체위협 카피,
  "Brain Rewiring %/치료" 의사과학 정량화, 우주·네온·회전 구체 spectacle, 부정 감정 소셜 프루프.
- Covenant류: 감시형 accountability, 종교·도덕심판 톤, 자동 공유, "감시" 프레임.
- 남성성 과다·게임 경제(코인/뽑기/랭킹 강등)·완치 단정.
- generic beige / wellness diary drift(Calm/Finch/Bloom 류 무드 차용 금지).
- 노골적 표현·성인 사이트명/URL.
- 경쟁사 화면 복제.

차용(패턴 차원만, NoF 톤으로 재해석):
- 상태 반응형 캐릭터/수호 오브제(차분·따뜻, 전투·네온 금지).
- 큰 단일 위기 진입점(카피 전면 교체, "잠깐 멈춤"→호흡/타이머/대체활동).
- 충동≠실패 분리 로깅. 원인 enum(선택·비공개).
- 함께-차단 동료 초대(동의 기반·기본 비공개·비감시).
- 중립 스트릭(가혹 리셋 없음, 누적 여정 유지).

대상 화면(P0 우선순위, SCREEN_INVENTORY 기준 — 이번엔 무드/방향 비교용 대표 1~3개만):
홈 대시보드 / 위기 버튼 / 회복 / 캐릭터 홈 중에서 방향을 가장 잘 보여주는 화면.

제약:
- 한국어 카피·한글 타이포(Pretendard 계열) 전제.
- 최종 색·hex·고해상도 화면은 사람 승인 후. 이번엔 방향이 읽히는 무드 비교안.
- 입력 문서: DESIGN_DIRECTION.md, UX_RULES.md, COPY_POLICY.md, SCREEN_INVENTORY.md,
  CHARACTER_SYSTEM_SPEC.md, DIRECT_COMPETITOR_VISUAL_AUDIT.md, 본 브리프.

검수 통과 기준:
- 명상 앱 스크린샷 옆에 뒀을 때 "자기통제 앱"으로 구분되는가?
- §2 금지 톤(공포·수치·빨강·감시·종교·남성성·게임경제)이 0인가?
- 베이지/wellness diary로 회귀하지 않았는가?
- 동료성이 동의 기반·기본 비공개인가?
- 최종 색/UI를 임의 확정하지 않았는가?
```
