# CHARACTER_SYSTEM_SPEC.md

캐릭터 성장 시스템 — 리텐션·정서 지지 (처벌·pay-to-win 아님)

상위 문서: PRD §11 / 같은 라운드: UX_RULES.md, MONETIZATION_POLICY.md, COPY_POLICY.md
이 문서는 캐릭터 시스템의 추론·검수 기준이다. 캐릭터 디자인·에셋 생성은 하지 않는다.

---

## 1. 캐릭터 역할 (Character role)

캐릭터는 사용자를 평가하는 존재가 아니라, **자기통제 여정을 시각화하는 동료**다 (PRD §11.1).
목적: 리텐션(매일 열 이유) + 정서적 지지. 처벌·경쟁·과금 도구가 아니다.

---

## 2. 권장 세계관 (Recommended worldbuilding)

PRD §11.3 추천: **Option B + Option C 혼합** (NEEDS DECISION #12, #13 — 사람 확정 전까지 후보).
구체화 후보는 §13 **Ember Cat / 잉크 고양이 / 잔불 수호자** 참고 (handoff `COMPANY_PC_HANDOFF_20260526_EMBER_CAT.md` 기반, 미확정).

| 요소 | 방향 |
|---|---|
| 캐릭터 | 절제 환경을 함께 지키는 수호자/동료 |
| 홈 공간 | 사용자의 절제 루틴이 쌓이는 "내 절제 공간" |
| 성장 방식 | 스트릭·체크인·위기 극복·회복 플랜 |
| 꾸미기 | 캐릭터 외형·공간 테마·오브젝트 |

---

## 3. 캐릭터 성격 (Character personality)

- 차분하고 지지적. 동료 같은 거리감.
- 비난·평가·훈계하지 않음. 실패해도 곁에 머묾.
- 위기·실패 순간엔 짧고 따뜻하게. 성취 순간엔 담담히 인정.

---

## 4. 성장 원천 (Growth sources)

PRD §11.4. 행동의 결과로만 성장. 결제로 성장 금지.

| 사용자 행동 | 캐릭터 반응 | 보상 |
|---|---|---|
| 하루 유지 | 안정 상태 유지 | XP |
| 체크인 완료 | 오늘 감정 기억 | XP |
| 위기 버튼 사용 | 함께 버틴 메시지 | XP |
| 대체 활동 완료 | 에너지 회복 | XP |
| 실패 후 회복 플랜 | 다시 시작 응원 | 회복 배지 |
| 7일 스트릭 | 1단계 성장 | 진화/스킨 |
| 30일 스트릭 | 장기 성장 | 테마/칭호 |

> 성장은 스트릭 단독이 아니다. 입력은 체크인 누적·위기 버튼 사용 후 버틴 횟수·차단 세션 설정·회복 루프 완료·연속일+총 기록일을 함께 본다. 스트릭이 끊겨도(실패/재발) 단계는 내려가지 않는다(§5). 상세 후보는 §13.

---

## 5. 성장 단계 (Growth stages)

- character_stage는 predefined_enum (PRD §18).
- P0: 기본 단계 + XP 누적 표시.
- 진화/단계 전환은 P1 (REQ-012).
- 강등 없음. 실패로 단계가 내려가지 않는다 (PRD §11.5).

---

## 6. 실패 상태 처리 (Failure state handling)

PRD §11.5.

| 금지 | 허용 |
|---|---|
| 캐릭터 죽음 | 캐릭터가 쉬는 상태 |
| 벌점 | 회복 미션 |
| 성장 단계 강등 | 오늘의 회복 기록 |
| 비난 메시지 | 재시작 응원 |
| 공개 실패 알림 | 사용자 선택 기반 비공개 기록 |

---

## 7. 보상 규칙 (Reward rules)

- 보상=행동의 결과. "운/뽑기"가 아니라 행동 누적.
- 위기 버튼 사용도 보상 대상 (사용을 깎지 않음).
- 보상 수령 화면에 "한 번 더 뽑기"·과금 압박 금지 (COPY_POLICY §11).

---

## 8. 꾸미기 규칙 (Customization rules)

- P1: 캐릭터 외형·공간 테마·오브젝트 (REQ-013).
- 꾸미기는 미적 선택이지 능력·성공률에 영향 없음.
- 유료 꾸미기는 P2이며 **비랜덤 단품/테마만** (MONETIZATION_POLICY).

---

## 9. 과금 경계 (Monetization boundaries)

PRD §11.3, §12.3 기준. 금지:
- 캐릭터 사망/실패 처벌
- 전투력·능력치·의지력 점수
- 유료 성장 부스팅
- 유료 스트릭 복구 / 실패 삭제
- pay-to-win 효과 (절제 성공·차단 해제를 돈으로)

단계:
- **P0**: 기본 캐릭터 성장 (행동 기반 XP).
- **P1**: 진화 / 꾸미기.
- **P2**: 유료 코스메틱 — 검증 후, 비랜덤만.

## 13. Ember Cat 후보 — 시각·성장 구체화 (NEEDS DECISION, 미확정)

출처: `docs/handoff/COMPANY_PC_HANDOFF_20260526_EMBER_CAT.md`. 세계관(§2 Option B+C)의 **구체화 후보**이며 사람 확정 전이다(NEEDS DECISION #12, #13). 최종 디자인·색·에셋 아님. DESIGN_RESET_BRIEF의 `Ink & Ember`(후보 A — Quiet Shield 다크) 톤과 정합.

**컨셉**
- "어두운 잉크 공간에서 작은 고양이 수호자가 내 절제 공간을 함께 지킨다."
- 이름 후보: Ember Cat / 잉크 고양이 / 잔불 수호자 / 조용한 동료.

**시각 원칙**
- 검은 잉크 실루엣의 작은 고양이. 앰버(ember) 포인트는 눈·꼬리 끝·숨결·주변 잔불에만 소량.
- 귀엽지만 유치하지 않게. 큰 눈·볼터치·과한 표정 금지.
- OPAL gem mascot은 "하나의 상징 오브제가 앱 전체 기억점이 된다"는 **구조만** 참고. 복제 금지(DIRECT_COMPETITOR_VISUAL_AUDIT §2.2).

**성장 표현 (단계 후보)** — §4 입력의 시각화. 캐릭터가 강해지는 게 아니라 **사용자 절제 공간이 안정**되는 것.
1. 작은 잉크 고양이 — 눈빛만 작게 켜짐
2. 곁에 작은 잔불이 생김
3. 고양이 주변 공간이 안정됨
4. 꼬리 끝 빛이 길어짐
5. 절제 공간/방/결계가 완성됨

**Ember Cat 절대 금지** (§5·§6·§9 상속 + 구체화)
- 동글동글 아기고양이 펫게임화, 큰 눈·볼터치·과한 표정
- 실패/재발 시 울거나 죽거나 퇴화
- 전투력·레벨업·랭킹 강등, 확률형 뽑기
- 스트릭 복구권·실패 삭제권·차단 해제권
- 사용자를 감시·판단하는 톤

**코스메틱 예시** (직접 구매형·성과 무관·비랜덤, §8·MONETIZATION_POLICY)
